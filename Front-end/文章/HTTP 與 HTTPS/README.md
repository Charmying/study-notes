# HTTP 與 HTTPS

在瀏覽網頁時，網址列開頭的 `http://` 與 `https://` 牽涉到網站的安全性與資料的加密保護。

<br />

## HTTP (HyperText Transfer Protocol，超文字傳輸協定)

HTTP 是一種在網頁瀏覽器與伺服器之間進行資料傳輸的通訊協定。當在瀏覽器輸入一個網址 (例如：`http://example.com`)，瀏覽器就會透過 HTTP 向該網站的伺服器請求資料，然後伺服器回傳 HTML、CSS、JavaScript 等內容給瀏覽器進行顯示。

### 特點

- 資料為明文傳輸：HTTP 所傳輸的資料都是「未加密」的純文字，任何攔截者都能看到傳送的內容。

- 連線速度較快：由於沒有加密與解密的過程，HTTP 的傳輸速度理論上會比 HTTPS 快一些 (但差異微乎其微)。

### 風險

- 容易遭到攔截與竊聽：像是輸入帳號密碼或信用卡號碼時，駭客可以透過「中間人攻擊 (Man-in-the-Middle, MITM)」竊取資訊。

無法驗證伺服器真偽：可能被假冒網站欺騙，造成個資外洩。

<br />

## HTTPS (HyperText Transfer Protocol Secure)

HTTPS 是在 HTTP 的基礎上，加入了 SSL/TLS 加密協定的「加密版 HTTP」，能夠保障與伺服器之間傳輸資料的機密性與完整性。

當看到網址是以 `https://` 開頭，通常瀏覽器也會顯示一個「🔒鎖頭圖示」，代表這個網站使用了加密連線。

### 特點

- 資料經過加密：使用 SSL (或現代的 TLS) 技術，保護傳輸的資料不被攔截。

- 具備憑證認證：網站需申請「SSL 憑證」，使用者可以確認該網站真實性，避免受騙。

- 提升網站信任度與 SEO 排名：Google 等搜尋引擎會將 HTTPS 視為加分項，有助於提高網站排名。

### 運作流程簡述

1. 使用者向網站發送 HTTPS 請求。

2. 伺服器回傳 SSL 憑證 (含公鑰)。

3. 瀏覽器驗證憑證是否合法、有效。

4. 雙方協商加密方式，交換密鑰。

5. 之後所有資料皆加密傳輸。

<br />

## HTTP 與 HTTPS 的比較

|  | HTTP | HTTPS |
| - | - | - |
| 資料傳輸方式 | 明文 | 加密 |
| 安全性 | 低 | 高 |
| 憑證認證 | 無 | 有 (需申請 SSL 憑證) |
| 常見用途 | 內部網頁、測試環境等 | 商業網站、登入頁、付款頁面等 |
| 搜尋引擎排名 | 普通 | 較佳 |
| 成本 | 無 (不需申請憑證) | 憑證可免費 (例如：Let’s Encrypt) 也有付費方案 |
| 瀏覽器顯示 | 無鎖頭圖示，有時顯警告訊息 | 顯示鎖頭圖示，提升信任感 |

<br />

## HTTPS 是否完全安全

雖然 HTTPS 提供了強大的加密與驗證機制，但這並不代表網站本身「100% 安全」。若網站程式碼有漏洞，或伺服器遭到入侵，即使是 HTTPS 網站也可能被利用來進行釣魚或散播惡意程式。

也就是說，HTTPS 解決的是「傳輸過程的安全」，但不代表網站內容本身沒有問題。

<br />

## 常見問題與補充說明

### 網站是否一定要使用 HTTPS？

是的。現代瀏覽器對 HTTP 網站會顯示「不安全」警告。即使是個人部落格或小型專案，也建議使用免費的 SSL 憑證 (例如：Let’s Encrypt) 來開啟 HTTPS。

### 使用 HTTPS 是否會拖慢網站速度？

雖然加密會增加一點點資源消耗，但透過 HTTP/2 協定的支援與瀏覽器快取優化，HTTPS 幾乎不會造成可感知的延遲，甚至在某些情況下速度更快。

<br />

## 總結

HTTP 與 HTTPS 的最大差異，在於是否具備資料加密與身份驗證機制。在現今網路環境中，HTTPS 已經是網站的標準配備，不只是保障用戶資料安全，更是維護網站信譽與搜尋排名的重要因素。

不論是使用者還是網站開發人員，都應該了解這些通訊協定背後的意義與安全性，這樣在網路世界中更加安心、安全互動。
