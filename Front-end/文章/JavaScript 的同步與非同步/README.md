# JavaScript 的同步與非同步

在學習 JavaScript 的過程中，同步 (Synchronous) 與 非同步 (Asynchronous) 是非常重要的觀念，決定了程式碼執行的方式，也影響如何處理事件、網路請求以及長時間運算的流程。

<br />

## 同步 (Synchronous)

### 程式碼會按照順序一行一行執行，前一行沒有完成，後一行就不會開始。

也就是說，同步程式碼是阻塞 (Blocking) 的。假設某一段程式需要花很久時間才能執行完，那麼後面的程式碼就要等到執行完後才能繼續。

範例

```javascript
console.log("1. 開始");

for (let i = 0; i < 3; i++) {
  console.log(`迴圈第 ${i + 1} 次`);
}

console.log("2. 結束");
```

執行結果：

```console
1. 開始
迴圈第 1 次
迴圈第 2 次
迴圈第 3 次
2. 結束
```

可以看到，程式碼就是照順序執行，沒有跳過、也沒有延遲。

<br />

## 非同步 (Asynchronous)

### 某些程式碼不會馬上執行完，而是先把工作交給瀏覽器或 Node.js 的背景任務處理，等完成後再通知主程式執行結果。

好處：不會阻塞程式碼的進行。即使某個任務需要花很久時間，其他程式碼仍然可以繼續執行。

### 常見的非同步操作

- 計時器：`setTimeout`、`setInterval`

- 網路請求：`fetch()`、`XMLHttpRequest`

- 事件監聽：`addEventListener`

- 檔案操作 (Node.js 環境)

範例

```javascript
console.log("1. 開始");

setTimeout(() => {
  console.log("2. 計時器執行");
}, 2000);

console.log("3. 結束");
```

執行結果：

```console
1. 開始
3. 結束
2. 計時器執行
```

雖然 `setTimeout` 設定了 2 秒，但程式沒有等待，而是直接往下執行「3. 結束」。等到 2 秒過去之後，計時器的回呼函式 (Callback Function) 才被執行。

<br />

## 為什麼需要非同步

假設要從伺服器抓資料，通常需要好幾百毫秒，甚至幾秒。若用「同步」方式執行，整個頁面就會卡住，使用者不能點擊、不能操作。

透過非同步，JavaScript 可以

- 先發出網路請求

- 同時繼續執行其他程式碼

- 等到資料回來，再透過回呼函式 (Callback Function)、`Promise`、或 `async`/`await` 來處理結果

- 這樣就能確保應用程式保持流暢，不會卡死。

<br />

## JavaScript 為什麼需要這樣設計

JavaScript 本質上是單執行緒 (Single-threaded) 的語言，也就是同一時間只能做一件事。若沒有非同步，當程式遇到需要等待的任務 (例如：網路請求、I/O 操作)，就會把整個執行緒卡住，使用者體驗會非常差。

因此，JavaScript 搭配事件循環 (Event Loop) 與任務佇列 (Task Queue)，來達成非同步的效果。這樣即使只有一個執行緒，也能同時處理多個事件，模擬出「看起來同時進行」的效果。

<br />

## 總結

- 同步 (Synchronous)：按照順序一行行執行，會阻塞後續程式。

- 非同步 (Asynchronous)：把任務交給背景處理，等完成後再通知主程式，不會阻塞。

- 應用場景：計時器、事件監聽、網路請求、檔案讀寫。

- 目的：讓 JavaScript 在單執行緒的限制下，依然能保持流暢，提升使用者體驗。
