# SSL 與 TLS

在現代網路通訊中，安全性是一項極為重要的考量，無論是線上購物、登入系統，還是傳輸敏感資料，都希望這些資料能在網路上被加密傳輸，防止駭客攔截或竄改，這是就會時常聽到 SSL (Secure Sockets Layer，安全通訊層) 與 TLS (Transport Layer Security，傳輸層安全性)，廣泛應用於網頁瀏覽、電子郵件、即時通訊等場景。

<br />

## SSL (Secure Sockets Layer，安全通訊層)

SSL 是一種加密協定，由網景公司 (Netscape) 在 1990 年代初期所提出。目的是在應用層與傳輸層之間建立一層安全機制，使資料在網路上傳輸時能被加密，防止中間人攻擊 (Man-in-the-Middle Attack) 或資料竊聽。

SSL 曾廣泛應用於 HTTPS 網站，也就是在瀏覽器網址列看到的「🔒」符號背後的技術。不過，由於早期版本的 SSL 存在多個資安漏洞，像是 SSL 2.0 和 SSL 3.0 都已被認為不再安全，因此現今主流已不再建議使用 SSL。

<br />

## TLS (Transport Layer Security，傳輸層安全性)

TLS 是 SSL 的繼任者，也是目前主流的安全通訊協定。TLS 由 IETF (網際網路工程任務小組) 所標準化，並在 SSL 3.0 的基礎上進行改進與強化。

TLS 的設計理念與 SSL 相似，目標同樣是保護網路上的資料安全傳輸，但提供更高的加密強度與更嚴謹的驗證機制。目前常見的版本包括：

TLS 1.0 (1999)：早期版本，已不再建議使用

TLS 1.1 (2006)：同樣過時，瀏覽器多數已移除支援

TLS 1.2 (2008)：目前仍被廣泛使用

TLS 1.3 (2018)：最新版本，速度更快、安全性更強，已逐漸成為主流

<br />

## SSL 與 TLS 的主要差異

|  | SSL | TLS |
| - | - | - |
| 全名 | Secure Sockets Layer | Transport Layer Security |
| 發展機構 | Netscape | IETF |
| 最後版本 | SSL 3.0 (1996) | TLS 1.3 (2018) |
| 是否安全 | 否，已被淘汰 | 是，TLS 1.2/1.3 為主流 |
| 加密演算法 | 較舊，存在漏洞 | 使用更現代、更安全的演算法 |
| 握手流程 | 較慢，效率低 | 簡化握手流程，提高速度 |
| 警示訊息 | 兩種 (警告、致命)，未加密 | 加密，種類更多 (例如：關閉通知) |
| 訊息驗證 | 使用 MAC 與 MD5 (已淘汰) | 使用 HMAC，提供更高安全性 |
| 密碼套件 | 支援舊演算法 (例如：如 RC4、MD5)，有漏洞 | 支援先進演算法 (例如：AES-GCM、ChaCha20) |

簡單來說，TLS 是更安全、更現代的 SSL 替代品。而實務上口頭或文件中常提到「SSL 憑證」或「SSL 連線」，實際上所指的往往是基於 TLS 協定的連線，這是一種慣用術語，但本質上是使用 TLS。

## TLS 的工作原理

### 握手過程

握手過程是建立安全連線的關鍵步驟

1. 用戶端發送 Hello 訊息：列出支援的 TLS 版本、密碼套件和壓縮方法。

2. 伺服器回應 Hello 訊息：選擇 TLS 版本和密碼套件，傳送數位證書 (包含伺服器名稱、憑證頒發機構 (CA) 資訊和公鑰)。

3. 用戶端驗證證書：確認證書由可信 CA 頒發且有效。

4. 密鑰交換：用戶端生成隨機會話金鑰，使用伺服器公鑰加密後傳送。

5. 伺服器解密：使用私鑰解密，獲取會話金鑰。

6. 生成對稱金鑰：雙方使用會話金鑰生成對稱加密金鑰，用於後續資料傳輸。

### 資料傳輸

- 加密：使用對稱加密 (例如：AES-256) 保護資料，效率高於非對稱加密。

- 完整性：使用 HMAC 確保資料未被篡改。

- 憑證：TLS 使用 X.509 憑證進行身份驗證，確保連線安全。

<br />

## TLS 的應用

- 網頁瀏覽：HTTPS (HTTP over TLS) 是網站安全標準，顯示鎖頭圖標 (例如：Google、Facebook)。

- 電子郵件：保護 SMTPS、IMAPS 等協議。

- 即時通訊：例如：WhatsApp、Signal。

- VoIP：Skype、Zoom 等。

- 其他：FTP、VPN 等。

TLS 憑證 (常稱 SSL 憑證) 由憑證頒發機構 (CA) 頒發，需安裝在伺服器上。Cloudflare 提供免費 TLS 憑證 (Cloudflare SSL)。

<br />

## TLS 的重要性

- 保護資料：防止駭客竊取敏感資訊 (例如：信用卡號、個人資料)。

- 提升信任：HTTPS 網站顯示鎖頭圖標，增加使用者信心。

- SEO 效益：Google 優先排名 HTTPS 網站，提升搜尋引擎表現 (iBest)。

- 瀏覽器要求：Google Chrome 等瀏覽器對非 HTTPS 網站顯示警告，影響使用者體驗。

<br />

## 總結

TLS 已完全取代 SSL，成為網際網路安全通訊的標準。隨著技術進步，TLS 1.2 和 1.3 提供更高的安全性和性能，建議網站和應用程式採用最新版本以確保資料安全。部署 TLS 憑證不僅保護使用者資料，還能提升網站信任度和 SEO 效果，是現代網路不可或缺的一部分。
