# Big O 表示法 (Big O Notation)

Big O 表示法是一種數學符號，用來描述「演算法在輸入資料量增加時，執行時間或記憶體使用量的成長趨勢」，尤其著重於最壞情況的效率，不代表實際執行秒數，而是幫助評估演算法是否具有良好的擴展性 (Scalability) 與效率 (Efficiency)，特別在處理大量資料時更顯重要。

### Big O 的用途

- 分析演算法的時間複雜度 (Time Complexity)

- 分析演算法的空間複雜度 (Space Complexity)

- 幫助選擇最適合的資料結構與演算法解法

- 比較兩個演算法在大資料下的效能差異，找出程式的效能瓶頸 (CPU 或記憶體)

- 判斷一段程式是否可擴展 (Scalable)

<br />

## 時間複雜度 & 空間複雜度

- 時間複雜度 (Time Complexity)：表示演算法所需的執行時間隨著輸入資料量 ($n$) 變化的情形。

- 空間複雜度 (Space Complexity)：表示演算法所需的記憶體空間隨著輸入資料量變化的情形。

<br />

## 常見的 Big O 類型

以下是從效率最高到最低的常見時間複雜度，由小到大排列

| Big O | 名稱 | 舉例程式 | 說明 |
| - | - | - | - |
| $O(1)$ | 常數時間 | `return arr[0];` | 無論輸入多大，執行時間固定不變 |
| $O(\log n)$ | 對數時間 | 二分搜尋 | 資料每次都減半 |
| $O(n)$ | 線性時間 | for 迴圈 | 輸入多大，時間就花多長 (每個元素走一次) |
| $O(n\log n)$ | 線性對數時間 | 快速排序、合併排序 | 分治法排序演算法的典型複雜度 (高效排序演算法)，例如：Merge Sort、Quick Sort 的平均情況 |
| $O(n^2)$ | 平方時間 | 雙層巢狀迴圈 | 每個元素與所有其他元素互動 (例如：氣泡排序、選擇排序。常見於巢狀結構或暴力法) |
| $O(2^n)$ | 指數時間 | 遞迴費波那契 (未最佳化) | 每次呼叫自己兩次，成倍增長 (資料稍多就爆炸性成長)，例如：遞迴解法的費波那契數列 |
| $O(n!)$ | 階乘時間 | 全排列 | 組合與排列問題 (非常低)，例如：排列組合的暴力解法，像是旅行推銷員問題 TSP |

<br />

## 時間複雜度範例

### $O(1)$：常數時間 – 陣列取值

```javascript
function getFirst(arr) {
  return arr[0];
}
```

- 不論陣列有幾個元素，這個函式只會做一次存取，時間固定

### $O(n)$：線性時間 – 遍歷陣列

```javascript
function printAll(arr) {
  for (let i = 0; i < arr.length; i++) {
    console.log(arr[i]);
  }
}
```

- 資料越多，執行次數就越多

### $O(n^2)$：平方時間 – 巢狀迴圈

```javascript
function printPairs(arr) {
  for (let i = 0; i < arr.length; i++) {
    for (let j = 0; j < arr.length; j++) {
      console.log(arr[i], arr[j]);
    }
  }
}
```

- 巢狀迴圈導致執行次數為 $n$ × $n$

### $O(\log n)$：對數時間 – 二分搜尋

```javascript
function binarySearch(arr, target) {
  let left = 0;
  let right = arr.length - 1;
  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
```

- 每次搜尋都把問題切一半

### 「非對稱巢狀迴圈」與「多變數」情況

```javascript
for (let i = 0; i < n; i++) {
  for (let j = 0; j < m; j++) {
    // ...
  }
}
```

- 若巢狀迴圈的次數不相等，需考慮不同變數，例如：外層 $n$、內層 $m$，則時間複雜度為 $O(n × m)$

<br />

## 如何判斷一段程式的時間複雜度

1. 找出重複結構 (例如：迴圈、遞迴)。

2. 確認輸入變數與其關係 (是否依據 $n$ 執行多次)。

3. 忽略常數、只看增長趨勢 (例如：$O(3n)$ → $O(n)$)。

<br />

## 空間複雜度範例

### $O(1)$ – 單一變數儲存總和

```javascript
function sumArray(arr) {
  let total = 0; // O(1)
  for (let i = 0; i < arr.length; i++) {
    total += arr[i];
  }
  return total;
}
```

- 時間複雜度：$O(n)$，空間複雜度：$O(1)$，只用了常數空間儲存 `total`

### $O(n)$ – 建立新陣列

```javascript
function createArray(n) {
  let result = [];
  for (let i = 0; i < n; i++) {
    result.push(i);
  }
  return result;
}
```

- 空間複雜度為 $O(n)$，因為建立了一個 $n$ 長度的新陣列

### 遞迴 $O(2^n)$ – 費波那契遞迴 (未最佳化)

```javascript
function fib(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
}
```

- 每次呼叫會觸發兩次新的遞迴，因此時間複雜度為 $O(2^n)$

- 遞迴深度也導致空間複雜度為 $O(n)$ (堆疊深度)，因為在最深的遞迴展開時，呼叫堆疊會保留 $n$ 個待回傳的函式。

### 遞迴空間

```javascript
// 空間複雜度為 O(n) 因為會有 n 層遞迴堆疊
function fib(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
}
```

- 在費波那契遞迴中，除了時間複雜度 $O(2^n)$，也會產生遞迴棧空間 $O(n)$

## Big O 的規則 (只保留最主要的增長項)

| 分析技巧 | 範例 |
| - | - |
| 去除常數 | $O(3n)$ → $O(n)$ |
| 只保留最高次項 | $O(n^2 + n)$ → $O(n^2)$ |
| 巢狀迴圈要相乘 | 兩層 for 迴圈 → $O(n × n)$ = $O(n^2)$ |
| 多段執行 → 取最大項 | $O(n)$ + $O(\log n)$ → $O(n)$ |
| 遞迴需推展遞迴關係式 | 最佳化費波那契 (動態規劃) → $O(n)$ |

<br />

## 演算法分析的意義

使用 Big O 的好處是

- 幫助比較不同演算法的效率

- 提前預測在「大量資料」情況下的效能

- 優化程式碼時，找出瓶頸所在

<br />

## 判斷效率的黃金法則

| 程式碼特徵 | Big O 類型 |
| - | - |
| 無迴圈、無遞迴 | $O(1)$ |
| 單層迴圈 | $O(n)$ |
| 巢狀迴圈 (兩層) | $O(n^2)$ |
| 每次將問題切一半 | $O(\log n)$ |
| 遞迴中每次呼叫兩次 (類似樹) | $O(2^n)$ |

<br />

## Big O 在面試與實務的應用

| 題目 | 時間複雜度 |
| - | - |
| 陣列中找最大值 | $O(n)$ |
| 判斷字串是否為回文 | $O(n)$ |
| 泡沫排序 (Bubble Sort) | $O(n^2)$ |
| 快速排序 (Quick Sort) | $O(n\log n)$ |
| 建立 Hash Map 並查詢 | $O(n)$ 建立 + $O(1)$ 查詢 |
| 費波那契第 $n$ 項 | $O(2^n)$ (遞迴) 或 $O(n)$ (動態規劃) |

### 實務情境

| 任務 | 差別 (低效 vs 高效) |
| - | - |
| 搜尋資料 | $O(n)$ vs $O(\log n)$ (二分搜尋) |
| 排序資料 | $O(n^2)$ vs $O(n\log n)$ |
| 查找字典中某單字是否存在 | $O(n)$ vs $O(1)$ (使用 Set) |
| 預測效能瓶頸 | 利用 Big O 分析找出效能重點 |

<br />

## 最好/平均/最壞情況

Big O 通常分析最壞情況，但在某些演算法中，也會評估

- 最好情況 (Best Case)：以 Bubble Sort 為例，若輸入資料已完全排序，則為 $O(n)$

- 平均情況 (Average Case)：快排的平均情況為 $O(n\log n)$

- 最壞情況 (Worst Case)： 若快速排序在已排序的輸入資料上執行，可能會退化為 $O(n^2)$。

    若採用隨機化選軸法或三點取中法 (Median-of-three)，則可有效降低退化的機率。


<br />

## 常見誤解與陷阱

| 常見誤解 | 正確觀念 |
| - | - |
| $O(n)$ 就一定很慢 | 要看資料大小與實作方式 |
| 只有最壞情況要分析 | 最好、平均、最壞情況都可分析 (視需求) |
| Big O 是實際的執行時間 | 只描述增長趨勢，不是秒數 |
| Big O 越小就一定跑得比較快 | 實際還需考慮常數與實作細節 |
| 多段程式的 Big O 要全部加起來 | 只需保留最大影響的項目 |
| $O(n^2)$ 很差，不能用 | 資料小時仍可接受，視情況而定 |

<br />

## 總結

Big O 並不表示實際執行時間，而是描述「趨勢」與「成長速率」，Big O 能幫助

- 寫出更快、更省記憶體的程式

- 建構穩健、可擴充的大型系統

因此，在設計演算法時，Big O 是評估可擴展性與潛在效能的重要指標，而非實際執行速度的絕對衡量。
