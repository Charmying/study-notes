# 反向代理

反向代理 (Reverse Proxy) 是網路架構中一種常見的技術，主要功能是位於使用者端和後端伺服器之間，接收來自使用者的請求後，將其轉發至後端伺服器進行處理，然後再將伺服器的回應傳回給使用者。使用者無需直接與後端伺服器互動，因為所有請求都會經過反向代理伺服器來進行處理。

反向代理伺服器的功能相當多樣，無論是在提升系統效能、加強安全性，或是進行負載平衡，都能發揮關鍵作用。

<br />

## 反向代理的工作原理

反向代理伺服器會作為中介，處理所有來自外部網路的請求，這些請求可能是網站內容的讀取、API 的存取，或是其他與伺服器的互動。

1. 使用者端發送 HTTP 請求至伺服器 (例如：瀏覽器要求一個網頁)。

2. 請求先到達反向代理伺服器，而不是直接到達後端伺服器。

3. 反向代理根據設定的規則，將請求轉交給適當的後端伺服器進行處理。

4. 後端伺服器回應處理結果 (例如：返回一個網頁、JSON 資料等)。

5. 反向代理伺服器接收後端伺服器的回應，並將結果轉發給使用者。

使用者感受到的僅僅是與反向代理伺服器在互動，完全不會知道後端伺服器的存在或其具體運作。

## 反向代理的優缺點

### 優點

- 負載平衡

    反向代理的主要功能之一就是負載平衡 (Load Balancing)，可以將來自使用者端的請求分配給多個後端伺服器，避免單一伺服器過載。這樣有助於提升整體系統的效能與穩定性，確保網站或應用程式能夠在高流量的情況下穩定運行。

    範例：一個擁有大量流量的電子商務網站可以透過反向代理，將購物請求分配給不同的伺服器群組 (例如：產品瀏覽伺服器、結帳伺服器等)，確保每個伺服器的負載是均衡的，避免單一伺服器過載崩潰。

- 提高安全性與隱私保護

    反向代理能夠隱藏後端伺服器的真實 IP 地址，這樣攻擊者無法直接對後端伺服器發動攻擊。此外，反向代理可以作為第一道防線，對進入的請求進行過濾，並阻擋潛在的惡意請求。

    範例：當有駭客試圖對某個網站進行 DDoS 攻擊時，反向代理可以幫助屏蔽部分惡意流量，避免後端伺服器直接遭受攻擊，提升整體系統的安全性。

- 內容快取 (Caching)

    反向代理可以快取靜態內容 (例如：圖片、CSS、JavaScript 檔案等)，減少後端伺服器的工作負載。快取功能可以加速網站的回應時間，並且提升使用者的瀏覽體驗。

    範例：假設一個新聞網站的首頁流量非常大，反向代理可以將頻繁訪問的靜態資源 (例如：圖片和影片縮圖) 快取起來。這樣當其他使用者再訪問同樣的內容時，反向代理能直接提供這些快取資源，不需要每次都向後端伺服器請求。

- SSL 移除

    SSL 移除 (SSL Offloading) 是反向代理的另一大優勢。反向代理可以集中處理所有的 SSL 加密與解密工作，後端伺服器只需負責處理解密後的數據，減輕了後端伺服器的負擔。

    範例：當在進行網上銀行交易時，所有數據都是透過 SSL 加密。反向代理可以負責這些加解密操作，確保交易數據的安全，而銀行的後端伺服器則專注於處理交易規則。

- 便於管理多個伺服器或應用程式

    反向代理能夠整合多個不同的後端伺服器或應用程式，讓外部使用者只需要通過一個統一的入口點進行訪問。這能簡化管理，並且提高系統的靈活性。

    範例：一個大型企業可能有多個網站和應用程式，通過反向代理，所有的服務都可以通過單一的域名來訪問。反向代理會根據請求的路徑或域名，將請求轉發到正確的後端伺服器。

- 提供彈性路由與伺服器虛擬化

    反向代理可以根據不同的 URL 路徑或域名，將請求路由到不同的後端伺服器或服務。例如：一個企業可能同時運行多個子系統，例如：產品服務、用戶服務等，反向代理能根據請求的特性將其分發到相應的伺服器。

    範例：假設企業網站有兩個子系統

    - `/users` 路徑處理用戶管理

    - `/products` 路徑處理產品展示

    反向代理可以根據請求的路徑將其分別發送到處理用戶和處理產品的不同後端伺服器，讓不同系統可以在同一入口下無縫協作。

### 缺點

-  單點故障風險

    雖然反向代理能分散後端伺服器的負載，但本身可能成為單點故障 (Single Point of Failure，簡稱：SPOF)。若反向代理伺服器出現問題，那麼所有經過請求都會受到影響，整個系統的可用性可能會因此下降。

    解決方法：為了解決這個問題，可以使用多個反向代理伺服器搭配高可用性架構。例如：伺服器叢集或熱備份，來避免反向代理成為單點故障。

- 額外的網路延遲

    由於反向代理伺服器介於使用者端與後端伺服器之間，這樣會增加額外的網路延遲。特別是在處理大量請求或是複雜的業務時，這種延遲可能會對整體系統的效能產生不利影響。

    解決方法：最佳化反向代理伺服器的配置，使用高效的快取機制，並搭配 CDN 技術，可以有效降低這種網路延遲。

- 伺服器資源開銷

    反向代理需要消耗額外的硬體資源來處理快取、負載平衡、加密解密等操作。特別是在高流量的情況下，反向代理伺服器本身可能會變成瓶頸，進一步影響系統效能。

    解決方法：對於高流量的網站或應用程式，可以考慮使用多台反向代理伺服器來分擔流量，並定期監控反向代理的資源使用情況，確保系統不會因資源不足而受限。

- 複雜性增加

    在大型系統中引入反向代理後，網路架構會變得更加複雜。這可能會增加配置和維護的難度，尤其是當需要針對不同的應用程式、伺服器群組設定不同的轉發規則時，管理負擔會顯著上升。

    解決方法：使用自動化工具來配置和管理反向代理，並且在設計系統架構時保持簡潔、清晰的配置規則，能夠減少複雜性。

- 潛在的快取一致性問題

    當使用反向代理的快取功能時，可能會遇到快取一致性問題。例如：後端伺服器的數據更新了，但反向代理的快取並未及時刷新，導致使用者獲取到的資料是過期的。

    解決方法：設置適當的快取過期時間 (TTL)，並定期清理過期快取資料。此外，也可以搭配版本控制機制，來確保資料的即時性和一致性。

## 反向代理的應用場景

- 大型網站的負載平衡

    當網站流量巨大時，透過反向代理分配流量給多個伺服器，能夠提升網站的容錯率和處理效率。這種應用對於電子商務、新聞網站或社群平台等高流量網站尤其重要。

- 安全防護

    反向代理能夠有效保護後端伺服器的隱私，阻擋潛在的網路攻擊，並配合其他安全技術 (例如：WAF) 來過濾惡意流量。

- 快取與 CDN (內容分發網路)

    反向代理可以與 CDN 技術結合，提供全球性的快取節點，加速靜態內容的交付。例如：當使用者從不同國家訪問網站時，反向代理可以根據地理位置分發到最靠近的伺服器，提升使用者的瀏覽體驗。

- 多站點架構
反向代理能整合多個後端服務，讓使用者只需與單一的伺服器入口互動。例如：企業內部的不同應用系統可以通過一個反向代理伺服器來協作，降低管理複雜度。

## 總結

反向代理是現代網路架構中不可或缺的技術。反向代理的多功能性使其能夠適應各種不同場景的需求，從負載平衡到安全防護，從快取加速到多站點管理。透過正確的配置和運用，反向代理可以大幅提升系統的效能與安全性，並為後端伺服器減少許多不必要的負擔。
