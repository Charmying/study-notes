# eACH 簡介

ACH 圈存扣款系統簡稱 eACH (Enhanced Automated Clearing House)，是指提供扣款、入帳結果資訊即時交換，帳務仍以批次方式進行結算的系統。

民眾經由書面授權 (DDA) 或電子化授權 (eDDA) 完成指定業者的扣款授權約定後，業者即可依據授權約定內容，經由 eACH 進行服務或商品費用的扣款。

## eACH 與批次 ACH 比較

| | 批次 ACH | eACH |
| - | - | - |
| 確定交易結果 | 2 個營業日 | 即時 |
| 服務性質 | 批次性服務 | 24 小時全年服務 |
| 適用交易 | 適用帳單週期性交易 | 適用各種交易 |
| 每日清算次數 | 1 次 | 2 次 |

## ACH 代收代付業務簡介

ACH (Automated Clearing House) 代收代付業務 (又稱：媒體交換業務) 是台灣票據交換所發展完成的一個批量轉帳系統，利用電子作業處理方式，由機關團體、公司行號委託金融單位代付款項給收款人 (ACH 代付業務)，或向繳款人代收款項的機制 (ACH 代收業務)。

ACH 由代收銀行自收受者的銀行帳戶扣款給代收發動者或由代付銀行把發動者的款項撥付到收受者的銀行帳戶，一切交易均在發動者與收受者銀行帳戶轉帳。

ACH 代收代付業務是以定時、批次、淨額清算方式來處理大宗的跨行轉帳交易，首先由金融機構依發動者的申請，將轉帳內容經由網路傳輸至台灣票據交換所進行分類、結算，再由相關銀行提回轉帳。台灣票據交換所 ACH 代收代付作業每天定時辦理一次，即系統於每天早上開機，接受各提出行提出 ACH 代收代付資料，下午二點辦理分類結算後由各收受行提回處理，下午營業結束併同當日退票交換差額一起送中央銀行辦理清算，提回行並得利用夜間離峰時段辦理整批入帳或扣帳。

ACH 代收代付業務清算流程因代收、代付案件性質不同及考量交易風險而異。代收案件資料先由發動行提出，經交換所分類結算、收受行提回、扣款，次日再送中央銀行清算；代付案件資料則由發動行提出，經交換所分類結算，中央銀行當日清算後再由收受行提回入帳。

## 相關名詞定義

ACH：金融機構對於客戶申請辦理代收或代付案件的跨行轉帳交易，以網路傳輸或錄製媒體等方式，將轉帳資料傳送台灣票據交換所分類、結算後，送交中央銀行清算。

代收案件：金融機構應客戶的申請，代為向委繳戶收取款項的案件。

代付案件：金融機構應客戶的申請，代為向受款戶支付款項的案件。

委繳戶：委託參加單位代為扣款 (繳款) 者。

受款戶：委託參加單位代為入帳者。

提示交換：參加單位提出代收或代付案件的 ACH 資料，經台灣票據交換所分類、結算後，由其他參加單位提回。

退件交換：參加單位提出代收或代付案件交易未成功的 ACH 資料，經台灣票據交換所分類、結算後，由原提出參加單位提回。

發動者：指向參加單位申請發動媒體交換業務者，通常是企業或政府機構。

收受者：指透過發動者或逕向參加單位申請接受媒體交換業務者，通常是個人或企業。

發動行：指接受發動者申請，提示媒體交換業務的參加單位。

收受行：指接受收受者申請，收受媒體交換業務的參加單位。為收受者代為入帳者，稱入帳行；為收受者代為扣款者，稱扣款行。
