# Base64

Base64 是一種編碼方式，用來將二進制數據 (例如：圖片、文件、音訊、或任何二進制內容) 轉換成純文本格式。由於很多網路通訊或儲存系統只能處理文本數據，因此 Base64 編碼能讓圖片、文件、影片片段等非文本數據以文本字符的形式傳輸或儲存。這種編碼的主要作用是將二進制資料轉換為 ASCII 編碼，使這些資料能夠被安全嵌入到 HTML、JSON、URL 等文本環境中。

Base64 編碼的核心思想是將三個 8 位元組 (即 3 個 byte，共 24 位) 轉換成四個 6 位元組，再對應到一個可顯示的 ASCII 字符集。這樣儘管資料以文字形式表示，卻可以保持原始資料的完整性。

<br />

## 為什麼需要 Base64 編碼

電腦中的數據通常是以二進制形式表示，也就是 0 和 1 的組合。很多網絡協議 (例如：HTTP、電子郵件協議等) 是基於文本，這些協議在傳輸時會對特殊字符或某些非文本字符進行限制。若直接在這些環境中傳輸二進制數據，很可能會造成數據損壞或傳輸錯誤。因此需要將二進制數據轉換成一種僅包含文本字符的形式，而 Base64 就是一種常用的解決方案。

<br />

## Base64 的編碼過程和解碼過程

### 編碼過程

Base64 的編碼過程將二進制資料 (例如：圖像、文件、字串等) 轉換成純文本格式。

1. 將輸入數據轉換為二進制格式 (位元組)

    將要編碼的數據轉換為二進制位元序列。例如：若輸入的是字串 'Man'。

    - 'M' 的 ASCII 編碼是 77，二進制表示為：01001101

    - 'a' 的 ASCII 編碼是 97，二進制表示為：01100001

    - 'n' 的 ASCII 編碼是 110，二進制表示為：01101110

    因此，字串 'Man' 被轉換為以下 24 位的二進制數據

    ```text
    01001101 01100001 01101110
    ```

2. 將二進制數據 (位元組) 分成 6 位元組

    Base64 使用的是 6 位元來表示一個 Base64 字符，而不是標準的 8 位元。

    因此，將原始的二進位資料以 24 位為單位分組 (即每組 3 個 8 位元的字節)，並將其轉換為 4 個 6 位元的區塊。

    ```text
    010011 010110 000101 101110
    ```

3. 將每個 6 位元組轉換為 Base64 字符

    將每個 6 位元組當作索引，查找 Base64 字符表來對應的字符。

    Base64 字符表的內容包括

    ```text
    A-Z (26 個字母)
    a-z (26 個字母)
    0-9 (10 個數字)
    + / (兩個符號)
    ```

    - 010011 的十進位數值為 19，對應 Base64 字符集中的 'T'

	- 010110 的十進位數值為 22，對應 Base64 字符集中的 'W'

	- 000101 的十進位數值為 5，對應 Base64 字符集中的 'F'

	- 101110 的十進位數值為 46，對應 Base64 字符集中的 'u'

    因此，字串 'Man' 經過 Base64 編碼後，對應的編碼結果為：

    ```text
    TWFu
    ```

4. 添加填充字符 (如有必要)

    Base64 的編碼要求輸出的長度必須是 4 的倍數。若輸入數據的長度不是 3 的倍數，則會在編碼結果末尾添加填充字符 '='，以保證結果的長度達到要求。填充字符的數量取決於原始數據不足的位數。

	- 若只剩下 1 個字節，則會加兩個 '=' 。

	- 若剩下 2 個字節，則會加一個 '=' 。

    例如：若只編碼 'Ma' 這個字串，最後的 Base64 結果會是 TWE= (後面加上了一個填充符號 =)。

### 解碼過程

Base64 解碼則是上述編碼過程的逆向操作

1. 將 Base64 編碼數據分割為 4 個字符一組

    將 Base64 字串按每 4 個字符一組進行處理。例如：對於編碼結果 TWFu，將其分成 4 個 Base64 字符

    ```text
    T W F u
    ```

2. 將每個字符映射回 6 位二進制數據

    根據 Base64 字符集，將每個字符轉換回 6 位二進制數據

	- 'T' 對應的二進制數據為 010011

	- 'W' 對應的二進制數據為 010110

	- 'F' 對應的二進制數據為 000101

	- 'u' 對應的二進制數據為 101110

    結果將是以下 24 位二進制數據

    ```text
    010011 010110 000101 101110
    ```

3. 將這些 6 位數據重新組合為 8 位數據

    將 6 位的數據重新組合為 8 位元組，也就是 3 個字節的二進制數據

    ```text
    01001101 01100001 01101110
    ```

4. 將二進制數據還原成原始數據

    將這些 8 位元組的二進制數據還原為對應的 ASCII 字符，得到原始的字串 'Man'

	- 01001101 對應 'M'

	- 01100001 對應 'a'

	- 01101110 對應 'n'

    Base64 字串 TWFu 解碼後恢復為原始字串 'Man'。

5. 處理填充字符 (如有必要)

    在解碼過程中，若 Base64 字串包含 '=' 填充符號，則需要在解碼時忽略這些符號。填充符號只用來保證編碼字串的長度為 4 的倍數，不會對最終解碼結果產生影響。

## Base64 的應用場景

- 網頁嵌入圖片

    Base64 編碼最常見的應用之一就是在網頁中嵌入圖片，不需要單獨加載圖片文件。這種方式特別適合於小型圖片，因為可以減少 HTTP 請求次數，加速網頁的加載速度。

    ```html
    <img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..." alt="Base64 圖片">
    ```

    以上 img 標籤中，圖片的二進制數據被轉換為 Base64 編碼並直接嵌入到 HTML 中，這樣瀏覽器可以直接顯示圖片，不需要額外下載文件。

- 電子郵件附件

    在電子郵件中傳遞附件時，二進制文件也會被轉換為 Base64 編碼。這是因為電子郵件系統通常只支持文本格式，為了保證文件能夠正確傳輸，附件需要先進行編碼。當收件人收到郵件時，郵件客戶端會自動將 Base64 編碼解碼，還原成最初的文件。

- URL 中的數據傳遞

    某些情況下需要在 URL 中傳遞數據。由於 URL 的特殊字符限制，某些字符 (例如：空格、非字母數字字符等) 需要進行編碼處理。Base64 可以將這些數據編碼成一串合法的 URL 字符，這樣可以避免數據傳遞時發生錯誤。

- JSON Web Token (JWT)

    JWT 是一種用來在網絡中傳遞身份驗證數據的標準格式，其中的數據是以 Base64 編碼的形式儲存。當伺服器或客戶端需要驗證某個用戶的身份時，會從 JWT 中提取 Base64 編碼的內容，解碼後進行解析。

## Base64 的優缺點

### 優點

- 跨平台傳輸：Base64 將二進制數據轉換成文本，能夠在任何文本協議中進行傳輸，實現了跨平台兼容性。

- 適合小型數據嵌入：在 HTML、CSS 或 JSON 中嵌入小型二進制數據時非常方便，能夠減少 HTTP 請求次數。

- 簡單易用：編碼和解碼的過程相對簡單，且有多種程式語言支援 Base64 編碼和解碼函數。

### 缺點

- 編碼後的數據量增加：Base64 編碼會將原始數據擴大約 33%。例如：1 MB 的圖片經 Base64 編碼後會變成 1.33 MB，這樣會佔用更多儲存空間和傳輸帶寬。

- 安全性不足：Base64 只是一種編碼方式，不是加密，不提供數據保護功能，任何人拿到 Base64 字串都可以容易解碼出原始數據。因此，敏感信息不應僅使用 Base64 進行保護。

<br />

## Base64 的簡單範例

可以使用多種程式語言來進行 Base64 編碼和解碼。

```javascript
let str = "Hello World";

let encodedStr = btoa(str); // 將字串轉換為 Base64 編碼
console.log(encodedStr);    // SGVsbG8gV29ybGQ=

let decodedStr = atob(encodedStr); // 將 Base64 編碼轉回字串
console.log(decodedStr);           // Hello World
```

## Base64 與 QR Code 的關係

### Base64 與 QR Code 的應用場景

Base64 編碼的主要功能是將二進制數據 (例如：圖片、文件、或其他非文本數據) 轉換成純文本字符，QR Code 是一種將文字、網址、或其他數據編碼為二維條碼的方式。因此，在某些情況下可以利用 QR Code 將 Base64 編碼的數據進行傳遞或共享。

- 嵌入小型文件或圖片：若想通過 QR Code 傳遞圖片或文件，而不想使用 URL 來指向外部資源，可以將圖片或文件轉換為 Base64 字串，然後將該字串嵌入到 QR Code 中。接收者掃描 QR Code 後，可以解碼並還原這些數據。

- JSON Web Token (JWT)：若需要在 QR Code 中傳遞 JWT 等身份驗證數據，這些數據通常是 Base64 編碼，因此可以將完整的 JWT 字串嵌入 QR Code 中，以便進行身份驗證等用途。

- 加密數據傳輸：儘管 Base64 不是一種加密方式，但可以先將數據加密，然後將加密後的二進制數據轉換為 Base64 編碼，再通過 QR Code 傳遞。接收方解碼後可以進行解密操作。

### Base64 與 QR Code 的限制

- 數據大小限制

    QR Code 雖然可以承載大量的數據，但容量有限。根據 QR Code 的版本不同，最大容量大約可以承載 4296 個字母數字字符或 2953 字節的二進制數據。由於 Base64 編碼會將數據增大約 33%，因此需要注意數據總量。若數據太大，QR Code 的大小會增大，並且可能超出可掃描的範圍。

- 字串長度

    Base64 編碼的字串在 QR Code 中會占用較多的空間。若數據過於龐大，生成的 QR Code 會變得非常複雜，容易影響掃描設備的讀取精度。在這種情況下，通常更推薦使用 URL 指向外部資源，而不是直接將大量的 Base64 編碼數據放入 QR Code。

- 傳輸效率

    將數據轉換為 Base64 編碼後嵌入 QR Code，雖然可以讓數據以文本形式傳輸，但效率不如直接使用 URL 等方式。尤其是在需要傳遞大型文件或圖片時，使用 Base64 並不是最佳選擇。

### 如何將 Base64 編碼嵌入 QR Code

1. 編碼資料為 Base64

    需要將想嵌入 QR Code 的數據轉換為 Base64 字串。

    ```javascript
    let str = "Hello World";
	let base64Str = btoa(str); // 將字串轉換為 Base64
	console.log(base64Str);    // SGVsbG8gV29ybGQ=
    ```

2. 生成 QR Code

    可以使用各種工具或 library 來生成 QR Code，將 Base64 字串嵌入其中。

    ```html
    <!DOCTYPE html>
	<html lang="zh-TW">
	<head>
	  <meta charset="UTF-8">
	  <title>QR Code with Base64</title>
	  <script src="https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js"></script>
	</head>
	<body>
	  <div id="qrcode"></div>

	  <script>
		let base64Str = "SGVsbG8gV29ybGQ="; // Base64 編碼後的字串
		let qrCode = new QRCode(document.getElementById("qrcode"), base64Str);
	  </script>
	</body>
	</html>
    ```

    這段程式碼會在網頁上生成一個 QR Code，其中包含了 Base64 字串 SGVsbG8gV29ybGQ= (Hello World 的 Base64 編碼)。掃描這個 QR Code 後可以獲取這個 Base64 字串。

3. 解碼 Base64

    當用戶掃描 QR Code 並獲取 Base64 字串後，可以使用 JavaScript 的 `atob()` 函數來解碼，將其轉回原本的數據。

    ```javascript
    let decodedStr = atob(base64Str);
    console.log(decodedStr); // Hello World
    ```

### Base64 與 QR Code 的實際應用範例

- 嵌入圖片數據

    假設有一張小型圖片，並且希望將其嵌入 QR Code 中而不使用外部鏈接，可以將圖片數據轉換為 Base64，然後生成包含該數據的 QR Code。

    ```html
    <img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..." alt="Example Image">
    ```

    若圖片比較小，Base64 編碼後的字串可以直接嵌入到 QR Code 中。掃描後，使用者可以解碼 Base64 字串，並還原成圖片數據。

-  安全數據傳輸

    假設想通過 QR Code 傳遞一些敏感數據，可以先將數據進行加密，然後將加密後的數據轉換為 Base64，最後嵌入 QR Code 中。接收者在掃描 QR Code 並獲取 Base64 字串後，解碼並解密，即可得到原始的敏感數據。

Base64 可以與 QR Code 相結合，用於在純文本的 QR Code 中嵌入二進制數據。這種做好適合在傳遞小型文件、圖片、或加密數據。然而，由於 Base64 編碼會增大數據量，並且 QR Code 的承載容量有限，因此當數據量較大時，可以考慮使用 URL 指向外部資源來替代。

## 總結

Base64 是一種將二進制資料轉換為文本字符的編碼方式，廣泛應用於網頁開發、數據傳輸、資料加密等領域。透過 Base64 可以有效將非文本數據嵌入到文本環境中進行傳遞。而 QR Code 則能作為一種二維碼技術，用來傳遞各種數據，當 Base64 與 QR Code 結合時，可以將二進制數據 (例如：圖片、文件等) 以文本形式嵌入 QR Code，實現靈活的數據傳輸。不過，在數據量較大時，應謹慎選擇使用方式，以避免 QR Code 過於複雜，影響掃描效果。
