# Docker

Docker 是一個開源的容器化平台，用於開發、運行和管理應用程式。

Docker 容器是一種封裝格式，以標準格式封裝應用程式的所有程式碼與相依性，使其可以在運算環境間快速可靠執行。Docker 容器是一款熱門的輕量型、獨立的可執行容器，其中包含執行應用程式所需的一切，包括程式庫、系統工具、程式碼和程式實際執行。Docker 也是一項軟體平台，可讓開發人員快速建立、測試及部署容器化應用程式。

Docker 利用操作系統層的虛擬化來實現應用程式的隔離和資源的高效使用。與傳統的硬體虛擬化不同，傳統的硬體虛擬化需要在每個虛擬機上運行完整的操作系統。

## 容器化技術的基礎

Docker 容器是基於 Linux 的一項技術，利用了以下技術

- Namespaces：為每個容器提供獨立的系統視圖，包括執行、網路、掛載點等，確保了容器之間的隔離性。

- Control Groups (cgroups)：限制和隔離容器的資源使用，例如：CPU、記憶體、磁碟 I/O 等，使 Docker 能夠控制每個容器的資源分配。

- Union File Systems (UnionFS)：支持構建映像的分層檔案系統。常用的有 AUFS、OverlayFS 等，允許 Docker 映像在多層之間共享檔案，減少了冗餘。

## Docker 的核心概念

- Docker 映像 (Images)

    Docker 映像是應用程式和其依賴環境的靜態 Template。映像包含了應用程式運行所需的所有內容，例如：程式碼、運行時環境、系統工具、程式庫等。這些映像可以儲存在 Docker Hub (Docker 的公共映像註冊中心) 或私有註冊伺服器中。作為容器執行的軟體集合，包含建立能夠在 Docker 平台上執行容器的一組指示。影像不可變更，影像需要變更才能建立新影像。

    - 構建映像：開發人員可以使用 Dockerfile，其中定義如何從基礎映像構建新的映像，包括執行的命令、添加的檔案等。Dockerfile 是描述應用程式及其環境的一種有效方式。

    - 分層結構：Docker 映像採用分層結構，每一層代表映像的不同狀態。也就是說，若多個映像共享相同的基礎層，Docker 只需在硬碟上儲存一次，大大提高了儲存效率。

- Docker 容器 (Containers)

    容器是 Docker 映像的運行實例，提供了輕量級、獨立的運行環境，確保應用程式及其所有依賴在任何支持 Docker 的平台上都能一致運行。

    與提供硬體虛擬化的 VM 不同，容器會抽象「使用者空間」，提供輕量而操作系統層次的虛擬化。容器與其他容器共用主機系統的核心。在主機作業系統上執行的容器是封裝程式碼及其所有相依性的標準軟體單位，因此應用程式可以快速可靠從某個環境執行到另一個環境。容器不是永久的，會從映像檔中擷取。

    - 獨立性：每個容器都是相互隔離的，有自己的檔案系統、網路接口和執行空間。這種隔離性確保一個容器內的變化不會影響其他容器或主機系統。

    - 資源效率：與傳統虛擬機不同，容器共享主機系統的操作系統的核心，使其更輕量，啟動速度更快，可以在同一主機系統上運行更多的容器。

- Docker 引擎 (Engine)

    Docker 引擎是用於創建、運行和管理 Docker 容器的核心元件。

    Docker 引擎扮演支援各種 Windows 伺服器和 Linux 作業系統上容器的從屬端伺服器應用程式，包括 Oracle Linux、CentOS、Debian、Fedora、RHEL、SUSE 以及 Ubuntu。

    - Docker Daemon：這是運行在主機系統上的背景處理程式，負責管理容器的生命周期。

    - Docker CLI：這是一個命令行工具，用來和 Docker Daemon 通訊，提供創建、運行、停止和管理容器的命令。

- Docker Hub

    Docker Hub 是一個公共的 Docker 映像註冊服務，類似於 GitHub 是程式碼的登錄檔。開發人員可以從中下載官方映像或社群共享的映像，也可以上傳自己的映像供他人使用。Docker Hub 提供了映像版本管理、團隊協作等功能。

## Docker 的工作流程

1. 構建映像：開發人員編寫 Dockerfile，描述如何從基礎映像創建新的映像。使用 docker build 命令來生成映像。

2. 推送映像：將映像推送到 Docker Registry (例如：Docker Hub)，以便共享和重用。

3. 拉取映像：在需要運行應用程式的機器上，從 Docker Registry 拉取映像。

4. 運行容器：使用 docker run 命令啟動容器，並提供運行時的配置參數，例如：網路、port、環境變數等。

5. 管理容器：運行中的容器可以被停止、重啟、刪除，或者使用 docker exec 命令進入容器執行命令。

## Docker 的優缺點

### 優缺點

- 輕量級和高效能

    - 資源消耗低：Docker 容器不需要完整的操作系統，與虛擬機相比更輕量，啟動速度更快。

    - 快速啟動：由於容器只需要啟動應用程式和其必要的依賴，因此啟動時間比虛擬機短很多。

- 一致性和可移植性

    - 環境一致性：Docker 容器打包了應用程式及其依賴，確保應用程式在開發、測試和生產環境中都能一致運行。

    - 跨平台支持：Docker 容器可以在任何支持 Docker 的平台上運行。

-  簡化的部署和管理

    - 自動化部署：Docker 支持 CI/CD，允許開發人員自動構建、測試和部署應用程式。

    - 容易擴展：容器化應用程式可以輕鬆在不同的主機上水平擴展。

    - 快速恢復：Docker 支持映像的版本控制，開發人員可以快速回復到以前的版本。

- 支持微服務架構

    - 服務隔離：每個微服務運行在獨立的容器中，確保服務之間的獨立性和隔離性。

    - 靈活性：微服務可以獨立部署和擴展，不會影響其他服務。

### 缺點

- 安全性問題

    共用核心：由於容器共享主機系統的核心，可能導致潛在的安全風險。容器之間的隔離性不如虛擬機。

    - 權限管理：容器中運行的進程通常以 root 身份運行，可能會導致安全漏洞。

- 持久化儲存的挑戰

    - 數據持久性：當容器被刪除時，內部數據也會丟失。需要使用卷 (Volumes) 或外部儲存來實現數據持久性。

- 網路複雜性

    - 網路配置：Docker 的網路設置較為複雜，尤其是在多主機環境中，需要額外的工具 (例如：「Kubernetes，簡稱：K8s」) 來管理和編排容器的網路。

## Docker 與 K8s (Kubernetes) 的比較

Docker 容器代表開發及部署容器化應用程式。隨著容器化應用程式的數目持續增加，跨越多個伺服器部署非常多容器，操作這些容器也越複雜。而 K8s 提供協調、調整、管理及排程這些容器。K8s 是一個開放原始碼協調系統，可執行 Docker 容器與工作負載。K8s 可以幫助在移動擴展部署在多個伺服器上的多個容器時管理操作的複雜性。K8s 引擎自動協調容器的生命週期，將應用容器分佈在託管基礎架構上。K8s 會視需要快速上下擴展或縮減資源，可以佈建、排程、刪除以及監督容器的狀況。
