# 營業日、系統日和工作天的差別

在金融機構中，尤其是銀行，日期管理是一個複雜又重要的系統。這些日期系統幫助銀行管理和調節其金融業務，確保所有交易和業務活動都在規範內進行。

<br />

## 營業日 (Business Day)

### 營業日主要面向客戶服務，直接影響客戶交易

營業日是銀行對外開放，並提供各種金融服務，對於客戶來說是可以親自或透過電子方式進行金融交易的日子。銀行台灣的營業日通常是週一至週五，排除國定假日和週末。

- 營業日的特點

    - 客戶服務

        - 營業日是銀行提供面對面服務的日子。客戶可以在銀行分行辦理各種金融服務，例如：開戶、貸款、投資諮詢等。

        - 銀行工作人員在這些日子會處理客戶的需求，解決問題並提供導引。

    - 交易處理

        - 大多數銀行的交易 (例如：存款、提款、轉帳等) 在營業日處理並立即生效。

        - 一些交易可能需要在特定的截止時間之前完成，以便在當天進行結算。例如：匯款交易可能需要在下午三點前完成，否則將順延至下一個營業日。

    - 系統維護

        - 銀行的資訊系統通常在非營業日進行維護，盡量減少對客戶的影響。

- 營業日使用範例

    - 週一 (營業日) 前往銀行存款，這筆交易將在當天或次一營業日處理。

    - 於週五下午提交貸款申請，由於時間接近銀行下班，該申請可能在週一處理。

<br />

## 系統日 (System Day)

### 系統日主要用於內部操作及交易處理，影響內部結算及報表

系統日是銀行內部系統所設定的日期，用於交易處理、帳務結算、報表生成等功能。系統日通常與營業日同步，但有時會與營業日不同，例如：系統維護或批次處理，可能會出現不同步的情況。

- 系統日的特點

    - 交易記錄

        - 所有的交易，不論是否在營業日發生，均需有一個系統日來標記其實際的處理時間。

        - 例如：客戶於週六透過 ATM 進行的提款操作，可能會在當日的系統日記錄，但實際結算和清算可能會順延至下一個營業日。

    - 批次處理

        - 系統日用於批次處理銀行內部交易，例如：利息計算、帳戶結餘更新等。

        - 這些批次處理通常在非營業日進行，特別是在週末夜間，確保在下一個營業日開始時，所有數據都是最新的。

    - 對帳與結算

        - 系統日對於銀行內部的帳務平衡和外部的結算至關重要，所有金融機構都必須確保其帳目的準確性和一致性。

        - 系統日會影響到銀行的風險管理策略，特別是在資金流動和信用風險管理方面。

    - 調整靈活性

        - 系統日可以被暫時調整以配合特定作業需求，例如：會計年度結算或系統升級。

- 系統日使用範例

    - 系統維護

        - 銀行決定在週六進行系統維護，雖然非營業日，但系統日仍可能被設定為前一營業日 (週五) 以便於交易結算。

    - 帳務結算

        - 月底結算時，銀行可能會將系統日暫停在月末日期來進行全面帳務對帳，即便當時實際日期已經進入下一個月。

    - 跨日交易

        - 客戶在週五晚上透過網路銀行進行的交易可能會在週六的系統日進行初步記錄，但在下週一營業日完成實際處理。

    - 利息結算

        - 銀行可能在每月月底進行利息結算，這一過程需要在一個特定的系統日完成，確保在下個月開始時，客戶帳戶的利息和餘額是正確的。

## 工作天 (Working Day)

### 工作天主要涉及員工的出勤及排班，影響人資管理

工作天是指銀行員工的實際上班日，並不等於營業日，取決於銀行內部需求及員工安排。工作天對於計算員工工時、薪資、假期等具重要性。

- 工作天的特點

    - 員工排班：工作天涉及銀行員工的排班、值班安排及假期規劃。

    - 內部作業：即便在非營業日，某些內部部門 (例如：IT、風控) 可能需要工作以進行必要的系統維護及交易監控。

    - 法律依據：工作天的定義通常受到勞動法規的影響，包括最低工時及加班計算等。

- 工作天使用範例

    - 在週末 (非營業日) 某銀行的 IT 部門仍需進行系統升級，因此這些員工的工作天包括該週末。

    - 銀行客服的標準工作天是週一至週五，但因銀行政策可能在某些週六提供特定服務。

## 其他日期概念

### 結算日 (Settlement Date)

結算日指金融交易中實際資金或證券轉移完成的日期。這個日期是交易雙方達成交易後，實際履行合約義務的日子

- 結算日的特點

    - 時間差異

        - 交易日期和結算日可能不一致。某些交易可能在達成後需要幾個營業日才能完成結算。

    - 影響

        - 結算日的延遲可能會影響資金流動和投資策略。

- 結算日使用範例：股票交易通常在交易日後的兩個營業日 (T+2) 完成結算。

### 交割日 (Delivery Date)

交割日指合約中標的資產或商品實際交付的日期。通常適用於期貨和期權等金融衍生產品。

- 交割日的特點

    - 合約履行： 在交割日，合約的雙方必須完成資產的交付和支付。

    - 風險管理： 交割日的選擇對於投資者的風險管理非常重要。

- 交割日使用範例

    - 某些期貨合約可能要求在合約到期後的一個特定日期進行實物交割。

### 截止日 (Cut-off Date)

截止日是指金融交易或報告必須在某個特定日期之前完成的日期。

- 截止日的特點

    - 時間限制： 許多金融操作 (例如：轉帳、報稅等) 需要在截止日之前完成，避免罰款或延誤。

    - 對應措施： 銀行通常會設立提醒系統，確保客戶和內部流程在截止日前完成必要操作。

- 截止日使用範例

    - 信用卡賬單的繳款截止日通常在每個月的某一特定日。
