# 雜湊 (Hashing)

雜湊 (Hashing) 是一種在電腦科學中被廣泛應用的技術，特別是在資料處理、加密、資料庫管理等領域。其主要功能是透過某種運算方式，將任意長度的輸入 (例如：文字、檔案等) 轉換為固定長度的輸出，這個輸出即為「雜湊值」或「雜湊碼」 (Hash Value)。透過這個過程能夠快速比較、驗證資料的完整性，或者進行高效查找操作。

雜湊函數是一個不可逆的演算法，也就是說，一旦資料經過雜湊處理，就很難從結果回推原始資料。因此，雜湊特別適合用於密碼學、數位簽章以及資料完整性的檢查。

<br />

## 雜湊的基本概念

- 雜湊函數 (Hash Function)

    雜湊函數是一個將輸入資料轉換為固定大小輸出的演算法。例如：輸入一段字串，經過雜湊運算後，會生成一個固定長度的雜湊值。不同的雜湊函數會生成不同長度的結果，例如：SHA-256 會生成 256 位元的雜湊值。

    範例

    ```text
    輸入字串: "password123"
    SHA-256 結果: 
    ef92b778bafe771e89245b89ecbcfb614d8c1965d3dabc8cf21a1af76d9189a1
    ```

    這個雜湊值是固定長度的，不論輸入資料是長是短，SHA-256 產出的結果都是一樣的長度。

- 碰撞抗性 (Collision Resistance)

    理論上，兩段不同的資料可能會產生相同的雜湊值，這種情況被稱為「碰撞」，但是好的雜湊函數設計會讓這種情況發生的機率極低。碰撞抗性是雜湊函數的基本要求之一。當使用某種雜湊函數時，應該能夠保證大部分不同的輸入都會產生不同的雜湊值。

- 不可逆性 (Irreversibility)

    雜湊函數的一個關鍵特性就是不可逆性。一旦資料經過雜湊處理，就無法根據雜湊值反推出原始資料。與加密不同，加密資料是可以解密還原，但是雜湊無法。

    範例：將字串 "password123" 經過 SHA-256 雜湊後得到一個固定的雜湊值，但即使知道這個雜湊值，也無法得知原本的字串。

## 雜湊的應用

- 密碼儲存與驗證

    在現代應用程式中，密碼不應該以明文形式儲存在資料庫中。系統會將密碼經過雜湊處理後再儲存。當使用者下次登入時，系統會將輸入的密碼再次進行雜湊，並與資料庫中的雜湊值進行比對。這樣，即使資料庫被駭客攻擊，駭客拿到的也是雜湊值，而非實際的密碼。

    範例

    ```text
    使用者輸入: "password123"
	  資料庫儲存的 SHA-256:
	  ef92b778bafe771e89245b89ecbcfb614d8c1965d3dabc8cf21a1af76d9189a1
    ```

    當使用者下次登入時，系統會對輸入的密碼再次雜湊，然後與資料庫中這個值進行比對，若一致，登入就會成功。

- 資料完整性檢查

    當在網路上傳輸資料或文件時，可以使用雜湊來檢查資料是否被篡改。發送者可以生成資料的雜湊值，接收者在接收到資料後也進行雜湊運算，然後將結果與發送者提供的雜湊值進行比對。若雜湊值相同，那麼資料是完整且未被修改。

    範例

    ```console
    發送前的 SHA-256:
    3a5c23db351b310ebebcf4f582bc82d9f2e9ffab62d1e5d93b5e3eebeb2f8b0d
    ```

    接收者在下載到該檔案後，也生成同樣的雜湊值。若兩者相同，代表檔案在傳輸過程中沒有被修改過。

- 雜湊表 (Hash Table)

    在資料結構中，雜湊表是一種高效的資料存取方式。透過雜湊函數將資料映射到表中的位置，可以快速查找到特定的資料。

    範例：假設要儲存一些人的電話號碼，可以使用雜湊函數將名字轉換為數字，並將這些數字用作資料表的索引。這樣當要查找一個人的電話號碼時，只需要對名字進行雜湊，然後直接查找相應的位置即可，查找效率會比線性查找高很多。

- 數位簽章

    在數位簽章中，雜湊用來生成文件的「數位指紋」，這樣即使文件中某一個字元被修改，雜湊值就會完全改變，從而檢測出文件是否被篡改過。

<br />

## 雜湊與加密的區別

雜湊和加密經常被混淆，但在本質上是不同的技術。

- 加密：是一種可以還原的技術，資料被轉換成密文後，仍然可以透過解密過程恢復原始資料。加密主要用於保護資料的私密性。

- 雜湊：是不可逆的，一旦資料經過雜湊處理，理論上應該無法從雜湊值還原出原始資料。雜湊的主要目的是驗證資料的完整性和一致性，不是保護資料的私密性。

## 範例程式碼

以 JavaScript 為例，展示如何使用 SHA-256 雜湊字串

```javascript
async function hashString(message) {
  const encoder = new TextEncoder();
  const data = encoder.encode(message);
  const hashBuffer = await crypto.subtle.digest('SHA-256', data);
  const hashArray = Array.from(new Uint8Array(hashBuffer));
  const hashHex = hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
  return hashHex;
}

hashString("Hello World").then(console.log);
```

以上範例中，程式碼會將字串 `Hello World` 轉換成其對應的 SHA-256 雜湊值。

## 總結

雜湊技術在電腦科學的領域中非常重要，無論是在資料儲存、驗證、加密還是資料結構中都有廣泛應用。雜湊的高效性、不可逆性和碰撞抗性使其成為一個可靠的工具，特別是在需要快速查找和確保資料完整性的情況。通過理解和應用雜湊技術，可以更好保護數位資料並提升系統效能。
