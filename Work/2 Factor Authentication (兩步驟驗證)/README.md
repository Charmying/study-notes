# 2 Factor Authentication (兩步驟驗證)

兩步驟驗證 (Two-factor Authentication，簡稱：2FA) 是一種安全性措施，目的在透過要求用戶提供兩個不同類型的資訊來確認身份，可以顯著提升系統的安全性，有效保護用戶的帳戶和敏感資訊。

<br />

## 2FA 的原理

2FA 是一種強化安全的認證機制，要求用戶在登入帳戶時提供兩個不同的驗證因素證明身份。這兩個因素必須屬於不同類型，這樣即使其中一個因素被攻擊者獲得，另一個因素也能防止未經授權的存取。2FA 通常涉及以下三類因素中的兩種

- 你知道的東西 (Knowledge Factor)：通常是密碼或 PIN (個人識別碼)，是用戶需要記住的資訊。

- 你擁有的東西 (Possession Factor)：這是用戶擁有的實物，例如：手機、硬體令牌或 IC 卡，通常用來接收或生成一次性驗證碼。

- 你是什麼 (Inherence Factor)：涉及生物識別技術，例如：指紋掃描、臉部識別或聲音識別等，利用用戶的生理特徵進行身份驗證。

<br />

## 2FA 的運作流程

1. 輸入密碼

    在使用 2FA 時，用戶首先按照通常的方式輸入帳戶密碼。這是最基本的驗證層，屬於「你知道的東西」這一類。

2. 提供第二驗證因素

    在正確輸入密碼後，系統會要求用戶提供第二種驗證因素。這通常是用戶「你擁有的東西」或「你是什麼」中的一種：

    - 發送驗證碼：系統可能會將一次性驗證碼 (One-Time Password，簡稱：OTP) 發送到用戶的手機或電子郵件。用戶必須在登入頁面上輸入此驗證碼。

    - 使用認證應用程式：用戶可以在手機上的應用程式 (例如：Google Authenticator、Authy) 中獲取動態生成的驗證碼。

    - 生物識別驗證：一些平台支持使用指紋掃描或臉部識別等生物特徵來進行二次驗證。

3. 驗證並登入：當用戶成功提供第二驗證因素後，系統會驗證身份，允許用戶登入並存取帳戶。

<br />

## 2FA 的優勢

- 增強安全性

    2FA 為帳戶提供了額外的安全層，也就是說即使密碼被攻擊者破解或洩露，仍然無法登入，因為需要同時掌握用戶的第二因素。

- 防止帳戶被盜

    透過要求兩種驗證因素，2FA 有效防止未經授權的存取，保護用戶的個人資料、金融資訊和其他敏感資料不被竊取。

- 減少釣魚攻擊的威脅

    即便用戶在釣魚網站上誤輸入了自己的密碼，由於攻擊者無法獲得用戶的第二因素，仍然無法存取用戶的帳戶，從而降低了釣魚攻擊的成功率。

- 符合安全合規要求

    許多行業標準和法律規範 (例如：PCI DSS、GDPR 等) 要求公司和組織實施 2FA 以保護用戶數據。實施 2FA 有助於企業達到這些合規要求。

<br />

## 2FA 的常見實施方式

- 簡訊驗證碼

    這是最常見的 2FA 方式之一。用戶在登入時收到一個發送到手機的簡訊驗證碼，需在系統中輸入此驗證碼來完成登入。

- 認證應用程式

    使用認證應用程式 (例如：Google Authenticator、Authy) 生成的動態驗證碼。這些應用程式不需要網路連接即可生成驗證碼，增加了便利性和安全性。

- 硬體令牌

    一些高安全性環境中使用專用硬體令牌，這些設備會生成定期更換的驗證碼，需要用戶在登入時輸入。

- 生物識別技術

    例如：指紋識別、臉部識別或聲音識別等，這些技術提供了一種自然且有效的身份驗證方法，尤其是在行動設備上。

<br />

## 2FA 的挑戰

- 用戶體驗

    增加的步驟可能會導致一些用戶感到麻煩，尤其是在驗證過程中遇到問題時可能會影響使用體驗。

- 設備依賴

    若用戶的手機不見、沒電或無法存取，則會難以完成第二步驟驗證，這樣對用戶造成不便。

- 簡訊攔截風險

    雖然簡訊驗證碼是常見的 2FA 方式，但若攻擊者能夠攔截簡訊或通過 SIM 卡交換獲得用戶號碼，這樣還是會帶來安全風險。

- 生物識別漏洞

    生物識別技術在某些情況下也可能會被欺騙或錯誤接受不匹配的特徵。
