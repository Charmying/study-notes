# 中間人攻擊 (MITM)

在資訊安全領域裡，中間人攻擊 (Man-in-the-Middle Attack，簡稱：MITM) 是一種常見且危險的攻擊手法。

中間人攻擊顧名思義就是攻擊者在兩個合法的通訊端點之間，悄悄在中間成為「中間人」，偽裝成對方進行攔截、篡改或竊取資料。由於這種攻擊往往不易被即時發現，因此對個人隱私、網路交易與企業安全構成重大威脅。

<br />

## 中間人攻擊的運作原理

一般來說，當使用者 (Client) 要與伺服器 (Server) 進行通訊時，雙方會直接建立一條安全的連線。但若遭遇中間人攻擊，攻擊者會偷偷「站在」兩者之間，並做到以下兩件事

- 攔截通訊：攻擊者在使用者與伺服器之間取得所有傳輸的資料。

- 偽裝身份：攻擊者同時假扮成伺服器對使用者回應，又假扮成使用者對伺服器回應。

也就是說，使用者以為自己正在和伺服器安全對話，實際上卻把資料先交給了攻擊者，再由攻擊者轉交給伺服器，並且整個過程雙方可能都毫無察覺。

<br />

## 常見的中間人攻擊方式

- Wi-Fi 竊聽

    攻擊者會架設一個名稱類似的「假 Wi-Fi 熱點」，例如：`CoffeeShop_Free_WiFi`。使用者一旦連上，就會把所有流量交給攻擊者，密碼、訊息、交易資料都有可能被攔截。

- ARP 欺騙 (ARP Spoofing)

    在區域網路中，攻擊者會發送惡意的 ARP 對應，讓電腦誤以為攻擊者的裝置才是真正的閘道器，進而把所有流量交給攻擊者。

- DNS 欺騙 (DNS Spoofing)

    攻擊者竄改 DNS 回應，讓使用者以為自己正在連線到合法的網站，但實際上卻被導向惡意網站。

- HTTPS 攔截

    若使用者連線的網站沒有正確實作 HTTPS，攻擊者可以強制降級至 HTTP，或利用偽造的 SSL 憑證進行攔截。

- 電子郵件竊聽與篡改

    攻擊者攔截電子郵件傳輸，可能竄改收件人或內容，常見於金融詐騙 (例如：把匯款帳號換掉)。

<br />

## 中間人攻擊的風險與後果

- 隱私洩露：密碼、個人資料、信用卡號碼可能外洩。

- 交易被竄改：攻擊者可能修改轉帳金額、交易內容。

- 惡意植入：攻擊者能在網頁中植入木馬程式或釣魚表單。

- 企業資料外洩：若發生在內部網路，可能造成商業機密被竊。

<br />

## 如何防範中間人攻擊

- 使用 HTTPS 與 SSL/TLS

    確保所有傳輸經過加密，並且憑證必須來自合法的憑證頒發機構 (CA)。

- 避免使用不明 Wi-Fi

    在公共場所上網時，盡量避免連接免費、沒有密碼的 Wi-Fi，或是搭配 VPN 使用。

- 檢查網址與憑證

    登入網站前確認網址是否正確，並檢查是否有安全鎖頭圖示，以及 SSL 憑證是否合法。

- 使用雙重驗證 (2FA)

    即使帳號密碼被竊，攻擊者仍需額外的驗證才能登入，能降低風險。

- 企業防護措施

    - 使用入侵偵測系統 (IDS) 與防火牆。

    - 啟用 DNSSEC 防止 DNS 偽造。

    - 嚴格管理內部網路，避免 ARP 欺騙。

<br />

## 總結

中間人攻擊是一種隱蔽且高風險的攻擊手法，攻擊者透過攔截與偽裝身份，能竊取甚至竄改敏感資訊。這類攻擊可能發生在公共 Wi-Fi、區域網路，甚至是電子郵件往來中。要有效防範，必須依靠加密技術 (例如：SSL/TLS)、謹慎的上網習慣，以及企業級的網路安全防護。

唯有同時強化技術措施與使用者警覺，才能降低中間人攻擊的威脅，確保通訊過程的真實性與安全性。
