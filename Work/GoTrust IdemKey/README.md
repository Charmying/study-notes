# GoTrust IdemKey

<img src="https://github.com/user-attachments/assets/73752cdb-0cab-4943-a807-92b47bf7bfb7" width="100%" />

在網路安全日益重要的時代，除了密碼之外，「實體安全金鑰」成為強化帳號安全、抵抗釣魚攻擊與帳戶被盜的關鍵設備。GoTrust (動信科技 GoTrustID Inc.) 就是在此領域投入的公司之一，其旗下一款主打產品便是 Idem Key (動信安全金鑰)。

在台灣，GoTrust 的產品也受到重視。根據資安界媒體與資安相關機構的報導，Idem Key 為通過資安院相關認證、可用於零信任架構的安全金鑰方案之一。

<br />

## 產品特色與技術規格

以下是 Idem Key 的主要特色與技術能力整理

| 項目 | 說明 |
| - | - |
| 協定支援 | 支援 FIDO U2F 及 FIDO2 標準，符合現代強化身分驗證方案 |
| 安全等級/認證 | 是全球已通過 FIDO2 Security Level 2 (L2) 認證的安全金鑰之一 <br /> 此外，GoTrust 官方也表示其使用的晶片通過 FIPS 140-2 Level 3 安全規範 |
| 防護能力 | 採用 IP68 防水防塵規格，可應對較嚴苛的環境條件 |
| 介面/連接方式 | 有 USB-A 或 USB-C 的版本 (視型號而定)，並支援 NFC 通訊，能與 iOS/Android 裝置互通 |
| 跨平台支援 | 可用於 Windows、macOS、iOS、Android 等作業系統平台 |
| 容量/資格儲存 | 根據使用手冊，Idem Key 可儲存多達 30 組 Resident Keys (即在金鑰上儲存帳戶驗證憑證) 等 |
| 尺寸/環境溫度 | 例如：Type-A 版本尺寸為 48.2 × 18.3 × 4.1 mm；Type-C 版本為 50.4 × 16.4 × 5 mm；操作溫度範圍約 0 至 40°C；儲存溫度可達 -20 至 85°C |

這些規格顯示 GoTrust 在安全金鑰設計上，兼顧協定兼容性、物理耐用性與跨平台應用。

## 功能與應用場景

以下是 Idem Key 在實際使用上可發揮的功能，以及適合的應用場景

### 主要功能

- 兩步驟驗證/多重身份驗證 (2FA/MFA)

    在支援 FIDO 的服務 (例如：Google、Microsoft、Facebook、GitHub、AWS 等) 註冊金鑰後，登入時插入或感應 NFC，即可完成安全驗證。GoTrust 官方資源中也列有不少支援 Idem Key 的服務設定指南。

- 密碼免登入/零信任架構下的登入

    在企業或系統架構中，Idem Key 可作為無密碼登入的一環，降低密碼被盜或重複使用的風險。

    特別是在 GoTrust 推出的 GoTrust ID 平台中，Idem Key 被整合為可用於「帳號登入」的實體驗證器。

- 支援離線登入/裝置登入

    在某些系統設計中，Idem Key 可用於本機裝置的登入 (視架構而定，不一定所有情況皆支援)。GoTrust 的說明中提到在某些情況下，即使裝置無網路，也能透過已註冊的安全金鑰來進行登入。

    不過要注意 GoTrust ID 本身的整體登入功能 (非僅安全金鑰) 通常針對企業環境設計，必須搭配其認證伺服器等架構。

- PKI/OTP/PIV 支援

    GoTrust 官方說明中指出 Idem Key 除了 FIDO2/U2F 之外，也支援 PIV、OTP、與 OpenSSL、Minidriver 等協定/工具整合。

    針對特殊組織或政府機構使用既有 PKI 基礎設施的情境，GoTrust 另有 Idem Key Plus 型號，強化 Web PKI 的整合。

### 適合的應用場景

- 企業資訊安全防護：員工登入公司內網、VPN、雲端服務時，可結合實體金鑰驗證以提升安全性。

- 零信任架構導入：在零信任安全設計中，實體金鑰作為強身分驗證的一環，減少對傳統密碼的依賴。

- 個人帳號安全強化：若關心各種雲端或金融帳號的安全，可把 Idem Key 作為第二層或主要驗證工具之一。

- 跨平台/行動設備應用：由於支援 NFC，可在手機或平板裝置上作為安全憑證使用 (前提是該服務支援 NFC 安全金鑰驗證)。

## 優點與可能限制/注意事項

### 優點

- 通過 FIDO2 Level 2 認證，在安全性上具備較高保證。

- 具備 IP68 防水防塵 設計，實體耐用性較佳。

- 多種介面：USB + NFC，方便在不同裝置上使用。

- 在台灣具有較好的本地認證與市場通路支持 (例如：通過資安院規範、被認定為安全金鑰方案)

- 與 GoTrust 的整體身份驗證平台 (GoTrust ID) 有整合，好處是在企業環境中部署時可比較順暢。

### 限制/風險 /注意事項

- 依賴服務端支援

    即使擁有 Idem Key，若希望登入的雲端/系統服務端並不支援 FIDO 金鑰驗證，那就無法使用。在採購/使用前需要確認目標服務是否有支援。

- 金鑰遺失或損壞風險

    實體金鑰若遺失、損壞或被竊，需要有備援方案 (備用金鑰、備份驗證方式等)。很多服務在註冊金鑰時也會要求備用方法。

- PIN 錯誤鎖定/重置問題

    若 PIN 輸入錯誤次數過多，金鑰可能會鎖定；重置後可能需要重新註冊所有帳號。GoTrust 的手冊有提到這類操作。

- 對於某些操作系統/環境支援限制

    雖然官方宣稱跨平台支援，但在某些極端或特殊環境 (例如：某些 Linux 發行版、較舊系統等) 可能有適配問題。

    例如：GoTrust ID 本身於企業環境中可支援 Linux 或 Mac，但在某些使用情境中並不保證完全支援所有功能。

- 軟體/管理平台依賴

    若只是買了 Idem Key，卻沒有搭配管理後端 (認證伺服器、使用者註冊流程、策略管理等)，那就只是個硬體金鑰。對於個人使用者來說，要達成完整的無密碼登入體驗，可能需要額外構建或採用支援金鑰登入的服務。

## 對比 YubiKey (與其他品牌) 市面選擇

若要把 Idem Key 與市場上較知名的安全金鑰 (例如：YubiKey、SoloKey、Feitian 等) 做比較，這裡有幾個觀察面向

- 有使用者在 Reddit 上討論指出，某些型號的 GoTrust Idem Key 僅支援 FIDO 協定，與某些 YubiKey 型號的功能比較接近。

- 儘管 YubiKey 是市面上知名度極高的品牌，但截至某些時候，YubiKey 未必都已通過 FIDO2 Level 2 認證 (視型號而定)。有評論認為 GoTrust Idem 在這方面具有未來性。

- 在成本、支援與本地服務整合方面，GoTrust 在台灣/華語市場可能具有較在地化的優勢 (例如：中文支援、通路物流、本地認證、政府/企業合作等)。

總之，選擇哪一款安全金鑰，應該根據在安全需求、服務支援、生態系統整合、本地支援等面向綜合評估。

## 使用與部署建議

以下是一些實務建議

1. 先確認目標服務/應用支援狀況

    在部署之前，先檢查要登入的服務 (例如：雲端平台、內部系統、SaaS、API) 是否支援 FIDO 金鑰登入。

2. 規劃備援/恢復機制

    實體金鑰可能遺失或鎖定，請規劃備用金鑰、備援驗證方式 (例如：TOTP、手機驗證、備用安全碼) 等。

3. 安全保存與管理

    實體金鑰的發放、回收、資產管理都要有嚴謹流程。若金鑰被盜或遺失，應該能立刻撤銷其在服務端的註冊權限。

4. 員工/用戶教育

    使用者需要理解如何正確使用金鑰、PIN 鎖定的風險、如何備援登錄等。

5. 搭配後端管理平台/認證伺服器

    對企業來說，金鑰只是驗證一環，還需要有後端系統來管理註冊、撤銷、策略控管、日誌審計等功能。搭配 GoTrust 的整體平台可能會更方便。

6. 測試環境先行試用

    在正式部署前，應先在測試環境中模擬各種失敗場景 (例如：金鑰失效、備援驗證、鎖定重置) 來驗證整體流程的可用性。
