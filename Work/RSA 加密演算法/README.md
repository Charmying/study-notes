# RSA 加密演算法

RSA 加密演算法是一種非對稱加密演算法，用來建立安全連線和創立數位簽名，在公開金鑰加密和電子商業等領域被廣泛使用。

<img src="https://github.com/user-attachments/assets/a9b07825-e935-4fe1-8aa5-0424a716716a" width="100%" />

RSA 是由 Ron Rivest、Adi Shamir 和 Leonard Adleman 在 1977 年發明，因此以他們三人的姓氏首字母命名。

RSA 的安全性是基於對極大整數做因式分解的難度，即使知道是兩個大質數的乘積，要分解出這兩個質數仍然有難度。也就是說，對一極大整數因式分解越困難，RSA 就越可靠，所以若有辦法找到一種快速因式分解演算法的話，那麼用 RSA 加密的訊息安全性就會下降。

## RSA 加密的運作原理

上面有提到，RSA 加密演算法是一種非對稱加密演算法，而非對稱加密演算法包含四個部分

1. 公鑰與私鑰的產生

2. (公開) 金鑰交換 (透過金鑰交換演算法)

3. 加密訊息

4. 解密訊息

## 公鑰與私鑰的產生

1. 選擇兩個質數 (𝑝 和 𝑞)：

    隨便選兩個大的質數 𝑝 和 𝑞，其中 𝑝 不等於 𝑞

    (2048 位元 RSA 加密使用 308 位數大小的質數。因為得出如此大的數字並不容易，所以需要進行質數測試)

2. 計算模數 (𝑁)：

    𝑁 = 𝑝 𝑞

    - 雖然計算很容易，但要算出是哪個數字相乘很困難

3. 產生公鑰 (𝑒)：

    RSA 公鑰由模數 (𝑁) 和公鑰指數值 (𝑒) 組成。在得到 𝑒 之前，需要先計算 𝑟 (𝑁 的總和)

    根據歐拉函數，求 𝑟 = 𝜑 (𝑁) = 𝜑 (𝑝) × 𝜑 (𝑞) = (𝑝 − 1) (𝑞 − 1)

    然後選擇 e (一個大於 1 但小於 𝑟，且與 𝑟 互質的任意整數)

    - 在 RSA 加密中，通常使用 65,537 作為 e

4. 產生私鑰 (𝑑)

    計算 𝑑 使得 𝑑 × 𝑒 ≡ 1 (mod 𝑟)。也就是說，𝑑 是 𝑒 在 𝑟 下的模反元素

### (𝑁,𝑒) 是公鑰，(𝑁,𝑑) 是私鑰。

## 金鑰交換

若想使用 RSA 加密訊息，就需要知道公鑰。分享公鑰可以透過任何可靠的方式，不一定要透過秘密方法。

<br />

## 加密訊息

用戶使用接收方的公鑰 (𝑁,𝑒) 對訊息 𝑀 進行加密，將 𝑀 轉換為一個小於 𝑁 的非負整數 𝑛，比如可以將每一個字轉換為這個字的 Unicode 碼，然後將這些數字連在一起組成一個數字，假如資訊非常長，可以將這個資訊分為幾段，然後將每一段轉換為 𝑛。然後將 𝑛 加密為被加密的訊息 𝐶。

𝐶 = 𝑛<sup>𝑒</sup> mod 𝑁

- 𝐶 可以用模冪演算法快速求出來。

- mod 代表取模操作 (modulo operation)，指的是一方除以另一方後剩下的餘數，例如：11 mod 3 = 2 (11 除以 3 等於 3 餘 2)。

- 若想傳送的訊息是數字，就將 𝑀 換成數字，此函數會包含所有缺失的部分。

- 若想傳送的是簡訊，需要先將訊息轉換成數字。

<br />

## 解密訊息

接收方使用自己的私密密鑰 (𝑁,𝑑) 對被加密的訊息 𝐶 進行解密。

## 解密原理

𝐶<sup>𝑑</sup> ≡ 𝑛<sup>𝑒⋅𝑑</sup> (mod 𝑁)

已知 𝑒𝑑 ≡ 1 (mod 𝑟)，即 𝑒𝑑 = 1 + ℎ𝜑 (𝑁)

那麼 𝑛<sup>𝑒𝑑</sup> = 𝑛<sup>1+ℎ𝜑(𝑁)</sup> = 𝑛 ⋅ 𝑛<sup>ℎ𝜑(𝑁)</sup> = 𝑛 (𝑛<sup>𝜑(𝑁)</sup>)<sup>ℎ</sup>

若 𝑛 與 𝑁 互質，則由歐拉定理得：

𝑛<sup>𝑒𝑑</sup> ≡ 𝑛 (𝑛<sup>𝜑(𝑁)</sup>)<sup>ℎ</sup> ≡ 𝑛 (1)<sup>ℎ</sup> ≡ 𝑛 (mod 𝑁)

若 𝑛 與 𝑁 不互質，則不失一般性考慮 𝑛 = 𝑝ℎ，以及 𝑒𝑑 − 1 = 𝑘 (𝑞 − 1)，得：

𝑛<sup>𝑒𝑑</sup> = (𝑝ℎ)<sup>𝑒𝑑</sup> ≡ 0 ≡ 𝑝ℎ ≡ 𝑛 (mod 𝑝)

𝑛<sup>𝑒𝑑</sup> = 𝑛<sup>𝑒𝑑−1</sup> 𝑛 = 𝑛<sup>𝑘(𝑞−1)</sup> 𝑛 = (𝑛<sup>(𝑞−1)</sup>)<sup>𝑘</sup> 𝑛 ≡ 1<sup>𝑘</sup> 𝑛 ≡ 𝑛 (mod 𝑞)

所以 𝑛<sup>𝑒𝑑</sup> ≡ 𝑛 (mod 𝑁) 得證。

## RSA 的應用範圍

- 資料加密：RSA 常用於保護敏感資料。透過公鑰加密，只有擁有相應私鑰的人才能解密。因此適用於保護需要高安全性的資訊傳輸，例如：銀行交易、個人身份資訊等。

- 數位簽章：數位簽章是 RSA 的一個重要應用。發送者使用私鑰對訊息進行簽章，接收者則使用公鑰驗證簽章真偽。這樣可以確保訊息完整性和來源可靠性，廣泛應用電子商務、法律文件和電子合約等領域。

- 密鑰交換：在網路安全通訊中，RSA 用於安全交換對稱加密演算法所需的密鑰。由於 RSA 的加密過程較慢，實際數據的加密通常使用對稱加密 (例如：AES)，而 RSA 則用於加密對稱加密的密鑰。

- 身份驗證：RSA 在身份驗證系統中用於確認使用者的身份。例如：使用者可以透過 RSA 加密自己的憑證或生物特徵資料，伺服器使用公鑰解密後驗證身份，這樣可以防止未授權的存取。

- 通訊安全：RSA 被廣泛用於 VPN 和 SSL/TLS 通訊協議中，確保網絡通訊的安全性。在這些應用中，RSA 用於加密會話密鑰，從而建立安全的通訊信道。

- 安全電子郵件：RSA 也應用在電子郵件的加密和簽章，確保郵件內容機密性和來源可靠性。PGP (Pretty Good Privacy) 和 S/MIME (Secure/Multipurpose Internet Mail Extensions) 是兩種常見的基於 RSA 的電子郵件安全技術。

- 區塊鏈和加密貨幣：在某些區塊鏈系統中，RSA 用於交易的簽章和驗證，確保交易的合法性、安全性和不可否認性，能幫助防止偽造交易和雙重支付等問題。

- 軟體防盜版：許多軟體公司使用 RSA 來防止未授權的軟體使用。透過 RSA，開發人員可以生成唯一的授權碼，並要求使用者在安裝或使用軟體時進行驗證。

- ATM 交易：許多自動提款機 (ATM) 使用 RSA 進行交易的加密，保護用戶的銀行訊息和交易數據。

- 電子支付系統：在電子支付系統中，RSA 用於保護交易訊息和用戶的支付細節。通過加密技術，可以防止交易過程中的敏感訊息被竊取或篡改，保護用戶的財務安全。

## RSA 演算法的漏洞

- 較弱的隨機數產生器

    RSA 演算法從選擇質數開始，而這第一步可能就是最易受攻擊的部分。若質數不夠隨機，就很容易因數分解並破解加密。

    較弱的隨機數產生器通常會在此處出錯，因此可以使用偽隨機數產生器來避免此漏洞。

- 錯誤的金鑰產生

    質數可能足夠隨機，但是數字太小或彼此太相近，就會是另一個問題。

    由於 RSA 演算法使用公鑰進行加密，因此金鑰大小對於防止因數分解至關重要。

    - 當數字較小時，找出質數 (𝑝 和 𝑞) 就更容易。

    - 若數字太相近，那麼私鑰 (𝑑) 就會相對較小。

    這就是為什麼建議最小金鑰大小為 2048 位元，而較新的 RSA 金鑰通常為 4096 位元。

## 針對 RSA 加密的攻擊

- 因式分解攻擊 (Factorization Attack)

    若不解決上述 RSA 演算法的漏洞，就可以利用漏洞進行因式分解攻擊。

    只有在沒有人能從模數 (𝑁) 中發現質數 𝑝 和 𝑞 時，RSA 加密才是安全的。若質數過於相近，或太小和不夠隨機，攻擊者就可以較容易對其進行因式分解。

- 旁路攻擊 (Side-Channel Attack)

    在旁路攻擊中，攻擊者會分析解密過程中收集到的額外資訊，而非破解加密金鑰。例如：檢驗演算法使用了多少電力，甚至是電腦在解密過程中所發出的聲音。

    時間攻擊也是旁路攻擊的一種。在 RSA 時間攻擊中，攻擊者會分析不同已知密文的解密時間，進而推導出私鑰 (𝑑)。

- 明文攻擊 (Plaintext Attack)

    若攻擊者比對出明文訊息的任何部分與密文，就可以進行明文攻擊。

    - 短訊攻擊：

        攻擊者可以存取明文訊息的某些部分，並對其加密來取得密文。就可以用其來推斷出明文訊息的其他部分。

        可以透過使用「填充方式」在加密明文訊息前，在訊息中包含額外資料，來避免此種明文攻擊。

    - 循環攻擊：

        攻擊者測試可能用於創立密文的排列操作。若成功就可以逆轉過程，從密文中取得明文訊息。

    - 公開訊息攻擊：

        這種情形比較少見，理論上來說，某些加密密文可能與原始明文訊息完全相同。若出現此種情形，攻擊者就可以推斷出加密訊息的其他部分，並進行公開訊息攻擊。

- 選擇密碼攻擊 (Chosen Cipher Attack)

    使用歐幾里得演算法來產生私鑰。由於這種演算法不是秘密，攻擊者可以使用拓展歐幾里得演算法從密文中取得明文訊息。這就是選擇密碼攻擊的原理。

## RSA 的優缺點

### 優點

- 高安全性

    - 大數分解困難：RSA 的安全性依賴於大數分解的計算困難性。當密鑰長度足夠大時，從公鑰推算出私密密鑰在計算上非常困難，因此 RSA 可以提供強大的安全保障。

    - 加密與簽名雙功能：RSA 不僅可以用來加密數據，還可以用來進行數字簽名，驗證數據的來源和完整性。

- 密鑰管理便利

    - 公鑰共享：公鑰可以安全公開分享，不需要安全渠道傳輸，簡化了密鑰管理和分發的過程。

    - 單向性強：即使擁有公鑰，沒有密鑰的情況下也無法解密，確保訊息的安全。

- 廣泛應用

    - 通用性：RSA 可以用於多種加密和簽名協議，例如：SSL/TLS (保障網絡通訊安全)、電子郵件加密 (例如： PGP)、VPN 以及數字簽名等。

    - 行業標準：RSA 是現代加密系統的基礎之一，廣泛應用於各行各業，並受到廣泛支持和信任。

- 非對稱加密的典型代表

    - 避免密鑰共享風險：在許多應用中，可以避免像對稱加密那樣的密鑰共享問題，減少被攔截和篡改的風險。

### 缺點

- 計算效率低

    - 加密和解密速度慢：與對稱加密演算法 (例如：AES) 相比，RSA 的加密和解密過程需要進行大量的模指數運算，速度較慢，不適合用於大數據量的加密。

    - 不適合長訊息的直接加密：由於運算效率較低，RSA 通常用於加密對稱密鑰或較小的數據塊，而不是直接加密大量訊息。

- 密鑰長度

    - 較長的密鑰需求：為了提高安全性，RSA 通常需要使用較長的密鑰 (例如：2048 位或更長)，增加了計算成本和存儲需求。

    - 隨密鑰長度增加，運算更慢：長密鑰提高了安全性，但也增加運算複雜度和計算時間。

- 潛在的量子威脅

    - 量子計算的影響：未來量子計算機的發展可能威脅到 RSA 的安全性。量子計算可以有效解決大數分解問題，可能使 RSA 在量子時代不安全。

    - 考慮量子安全方案：隨量子計算的進步，RSA 需要與其他量子安全加密技術一起使用，確保長期的數據安全。

- 潛在的攻擊風險

    - 弱質數問題：若選擇的質數 𝑝 和 𝑞 不夠大或不夠隨機，可能導致 RSA 密鑰的安全性降低。

    - 旁路攻擊：這些攻擊利用硬體的物理特性 (例如：耗電量、電磁洩漏等) 來推測私鑰。

    - 明文攻擊：攻擊者比對出明文訊息的任何部分與密文，就可以進行明文攻擊。

    - 選擇密碼攻擊：攻擊者可以使用拓展歐幾里得演算法從密文中取得明文訊息。
