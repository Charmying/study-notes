# FIDO2

在現代的數位生活中，社會大眾每天都要登入各式各樣的網站與應用程式，從網銀、社群媒體到工作平台，密碼已成為不可或缺的驗證方式。然而，密碼也成為資訊安全中最脆弱的一環，弱密碼、重複使用、釣魚攻擊與資料外洩，層出不窮。

為了解決這個長期存在的問題，FIDO 聯盟 (Fast IDentity Online Alliance) 提出了 FIDO2 標準，目標是讓使用者能夠「不再依賴密碼」，以更安全、方便的方式進行身份驗證。

<br />

## FIDO2 簡介

FIDO2 是由 FIDO 聯盟與 W3C 共同制定的開放式標準，用於在網頁與行動裝置上實現「無密碼登入」。

FIDO2 其實是兩個主要技術的組合

- WebAuthn (Web Authentication API)

    由 W3C 制定，讓網站能夠在瀏覽器中使用公開金鑰加密的登入機制。

    使得網站可以要求使用者註冊、驗證「安全金鑰」或「生物辨識裝置」(例如：指紋、Face ID、Windows Hello)。

- CTAP (Client To Authenticator Protocol)

    由 FIDO 聯盟制定，用於裝置之間的溝通協定。

    例如：使用者可以透過 USB 安全金鑰、NFC 或 藍牙裝置 進行驗證。

<br />

## FIDO2 的運作原理

FIDO2 採用 非對稱加密 (Asymmetric Cryptography) 機制，由「公開金鑰 (Public Key)」與「私密金鑰 (Private Key)」組成。

整體流程如下

1. 註冊階段 (Registration)

    - 使用者在網站上註冊帳號時，系統會要求綁定一個 FIDO2 裝置 (例如：指紋辨識器或安全金鑰)。

    - 裝置會在本地端產生一對金鑰

        - 私密金鑰：保存在使用者裝置中，永不離開裝置。

        - 公開金鑰：傳送給伺服器並儲存。

2. 登入階段 (Authentication)

- 當使用者再次登入該網站時，伺服器會送出一個隨機挑戰 (Challenge)。

- 使用者透過指紋、臉部辨識或插入安全金鑰來進行認證。

- 裝置會使用私密金鑰對挑戰簽章，並回傳簽章結果。

- 伺服器使用先前儲存的公開金鑰驗證簽章，若成功，即完成登入。

在整個過程中，私密金鑰不會上傳或暴露，大幅降低密碼外洩與釣魚攻擊的風險，這種設計讓認證過程在理論上達到「零知識傳輸」的安全等級。

<br />

## FIDO2 的優點

| 面向 | 說明 |
| - | - |
| 高安全性 | 採用非對稱金鑰加密機制，不需傳輸密碼或共用祕密，防止中間人攻擊與憑證竊取 |
| 方便易用 | 支援指紋、臉部辨識、PIN、實體金鑰等多種驗證方式，不必記密碼 |
| 跨平台支援 | 可在 Windows、macOS、Android、iOS 上使用，並被主要瀏覽器 (Chrome、Edge、Firefox、Safari) 支援 |
| 防釣魚設計 | 驗證時會綁定特定網站來源 (Origin)，攻擊者即使仿造網站也無法通過驗證 |

<br />

## FIDO2 的挑戰與限制

儘管 FIDO2 前景明亮，但在實務上仍面臨幾項挑戰

1. 使用者習慣轉換：許多使用者仍習慣密碼登入，對安全金鑰或生物辨識的信任度有限。

2. 裝置支援度差異：舊版瀏覽器或系統可能不支援 FIDO2 功能。

3. 服務端整合難度：開發人員需在伺服器端實作 WebAuthn 驗證規則，並妥善管理公開金鑰資料。

4. 裝置遺失問題：若使用者遺失註冊的安全金鑰，仍需備援機制 (例如：多重註冊或恢復程序)。

<br />

## FIDO2 的實際應用場景

Google、Microsoft、Apple 等大型科技公司已全面支援 FIDO2。

使用者可使用 Windows Hello、Android 生物辨識或實體金鑰 (例如：YubiKey) 登入帳號。

銀行與企業內部系統也逐漸導入 FIDO2，以取代傳統密碼登入或 OTP 驗證。

<br />

## FIDO2 與 Passkey

近年來，FIDO2 標準的實際延伸應用「Passkey (通行金鑰)」正逐漸普及。

Passkey 基於 FIDO2 標準，但更進一步簡化使用者體驗，讓金鑰可同步於雲端 (例如：iCloud Keychain 或 Google Password Manager)，

使用者只需以裝置解鎖即可登入，無需再插入實體金鑰。

不過，由於 Passkey 會透過雲端同步，因此雖然便利，但也必須依賴平台的安全機制與帳號保護措施。

<br />

## 總結

FIDO2 代表著身分驗證的重大轉折——從「輸入密碼」進化到「確認身分」，不僅提升安全性，更改善了使用體驗。

隨著 Passkey 的普及與各大平台的支援，未來「無密碼登入」將成為主流的身分驗證方式。
