# 預存程序 (Stored Procedure)

預存程序 (Stored Procedure，簡稱：SP)，概念是在資料庫管理系統 (DBMS) 中預先寫好並儲存 SQL 指令，指令可以是查詢、更新、新增、刪除等甚至更為複雜，可以進行條件判斷和流程控制，而且這些指令可以被重複調用，通常是用於執行一系列複雜的數據操作。

<br />

## Stored Procedure 的範例

以下是一個使用 MySQL 的簡單預存程序

```mysql
CREATE PROCEDURE GetCustomerOrders (IN customerID INT)
BEGIN
    SELECT OrderID, OrderDate, TotalAmount
    FROM Orders
    WHERE CustomerID = customerID;
END;
```

這個預存程序名為 `GetCustomerOrders`，接受了一個輸入參數 `customerID`，然後查詢並返回該客戶的所有訂單資料。

<br />

## Stored Procedure 的應用範圍

- 複雜查詢：當需要執行複雜的 SQL 查詢時，可以使用預存程序來封裝查詢流程，這樣可以簡化應用程式的指令。例如：根據多個條件篩選資料，進行多表聯接等。

- 批次處理：對於需要定期執行的批次操作 (例如：每天的資料備份、報表生成等)，使用預存程序可以自動化這些過程，減少人工干預。

- 業務規則封裝：將業務規則封裝在預存程序中，可以確保流程的一致性和集中管理。例如：訂單處理、庫存管理等業務規則。

- 資料驗證：在插入或更新資料時，可以使用預存程序進行資料驗證，確保資料的正確性和一致性。例如：檢查資料是否重複，是否符合業務規則等。

- 資料處理和轉換：需要對資料進行大量處理和轉換時，預存程序可以高效完成這些操作。例如：資料清洗、格式轉換等。

- 資料安全：使用預存程序可以限制使用者對底層資料表直接存取，增強資料的安全性。通過授予執行預存程序的權限，可以控制使用者只能通過預存程序存取資料。

- 交易管理：在需要保證資料一致性的操作中，預存程序可以封裝交易流程，確保所有操作全部成功或失敗。例如：銀行轉帳操作。

<br />

## Stored Procedure 的優缺點

### 優點

- 減少網路流量：預存程序在資料庫伺服器上執行，只需要傳送呼叫程序的命令，而不需要傳送大量的 SQL 語句和資料，因此可以減少網路流量。

- 安全性：通過預存程序可以控制使用者對資料的存取權限，只允許執行已批准的操作，從而提高了資料庫的安全性。

- 模組化：預存程序可以將複雜的業務規則分割，這樣可以更好管理和維護。

- 可重用：預存程序可以被多次呼叫，不需要每次都重寫相同的 SQL 語句。

- 預編譯：預存程序在創建時就已經被編譯並儲存在資料庫中，這樣在執行時不需要再次編譯，可以提高執行效率。

### 缺點

- Debug 困難：不像使用 Debug Mode 可以即時知道每行程式執行完會有什麼結果，預存程序中每句 SQL 都必須手動執行看看。

- 資料庫移植性差：不同資料庫管理系統的語法可能不同，從而影響可移植性。

- 版本控制困難：預存程序通常存在資料庫中，難以與程式碼一起進行版本控制。

- 修改成本高：若要修改資料表，必須用 IDE 全域搜尋所有有使用到這個欄位的預存程序，並對其做相應的修改。

- 非 Checked Exception (IDE & CI/CD 工具檢查不到)：用物件封裝的話，若改掉型態、移除變數或者函式的 Input/Output 改變了 IDE 會瘋狂報錯。
