# Session Key

在現代網路通訊與資安架構中，「Session Key (工作階段金鑰，或稱會話密鑰)」扮演著保護資料傳輸安全的重要角色。

Session Key 不僅確保使用者與伺服器之間的通訊內容不會被竊聽或篡改，也能在不影響效能的前提下維持加密的安全強度。

<br />

## Session Key 簡介

Session Key 是一種在通訊雙方建立連線時，由協定機制動態產生的暫時性加密金鑰，主要用途是用於單一工作階段 (Session) 內的資料加密與解密，在該連線結束後就會被捨棄。

- Session 代表一次通訊工作階段，例如：使用者與伺服器之間的一次登入、一次 API 請求、或一段連線存續時間。

- Key 是用於加密與解密資料的金鑰。

舉例來說：當使用者透過 HTTPS 連線至某網站時，瀏覽器與伺服器會先進行一次「金鑰交換」過程，用來產生一組獨立的 Session Key。此金鑰之後會被用來加密該連線期間的所有資料，例如：登入資訊、表單內容或 API 回應等。

<br />

## Session Key 的運作流程

以下以 HTTPS 通訊為例，說明 Session Key 的建立與使用過程

1. 連線初始化 (Handshake)

    - 使用者端 (例如：瀏覽器) 向伺服器發送連線請求。

    - 雙方交換公開資訊 (例如：伺服器的公開金鑰、支援的加密演算法)。

2. 金鑰交換 (Key Exchange)

    - 雙方透過非對稱加密機制 (例如：Diffie-Hellman 或 ECDHE) 協商出一組隨機數。

    - 根據這些隨機數，雙方各自運算出相同的 Session Key，伺服器與用戶端實際上不會直接傳送 Session Key。

3. 安全通訊開始 (Secure Session)

    - 之後的所有資料傳輸 (HTTP 請求與回應) 都會使用這組 Session Key 進行加密與驗證，除了加密外，Session Key 也可用於產生訊息驗證碼 (MAC)，確保資料完整性與真實性。

4. 會話結束 (Session Termination)

    - 當使用者登出或中止連線時，Session Key 即失效並被銷毀。

簡單來說，運作流程如下

1. Client → Server：發送連線請求  

2. Server → Client：傳送公開金鑰與加密參數  

3. 雙方透過 Diffie-Hellman 協商：生成 Session Key  

4. 後續通訊：使用 Session Key 加密資料傳輸

<br />

## Session Key 的特性

| 特性 | 說明 |
| - | - |
| 暫時性 | 只在當前連線期間有效，結束後即廢棄 |
| 唯一性 | 每次建立新連線都產生不同的金鑰 |
| 對稱加密 | 通常使用對稱演算法 (例如：AES、ChaCha20)，兼顧速度與安全性 |
| 保密性強 | 因為是動態生成的，即使被竊取也僅影響單一會話，不會危及其他連線。 |

<br />

## Session Key 與其他金鑰的差異

| 金鑰類型 | 用途 | 存續時間 | 加密方式 |
| - | - | - | - |
| Session Key | 用於會話期間的資料加密 | 短期 | 對稱加密 (AES、ChaCha20) |
| 公開金鑰/私密金鑰 | 用於身分驗證與金鑰交換 | 長期 | 非對稱加密 (RSA、ECDSA) |
| 主密鑰 (Master Key) | 用於從主密鑰派生多個 Session Key，常見於 TLS 的密鑰衍生機制 (Key Derivation Function，簡稱：KDF) | 中長期 | 依系統架構而定 |

<br />

## Session Key 實際應用範例

- HTTPS/TLS

    TLS 協定在交握階段 (Handshake) 中協商出 Session Key，用於加密後續 HTTP 資料。

- VPN (虛擬私人網路)

    每個 VPN 連線都會建立新的 Session Key，保護使用者端與伺服器之間的加密通道。

- 行動支付與網銀系統

    為避免交易資料被竊聽，每筆交易都會使用獨立的 Session Key。

<br />

## 安全性設計原則

- 金鑰不重複使用：每次 Session 都應建立新的 Session Key。

- 金鑰儲存保護：不得以明文形式儲存，應該使用安全記憶體區域或加密儲存機制。

- 安全銷毀：連線結束後立即清除金鑰記錄。

- 使用強隨機數生成器：避免可預測性，確保 Session Key 無法被推算。

<br />

## 總結

Session Key 是現代網路加密通訊中不可或缺的一環，兼具「效率」與「安全」兩大特點，透過短期、動態、對稱的金鑰設計，確保每一次的通訊都能在不暴露主密鑰的情況下安全進行。

可以將 Session Key 想像成「一次性密碼」的概念，每次建立新會話就生成新金鑰，用完即棄，確保資料安全與隱私不外洩。
