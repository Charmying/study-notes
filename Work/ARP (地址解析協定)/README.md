# ARP (地址解析協定)

在現代網路中，無論是家用網路還是企業內部網路，資料在不同裝置之間傳遞時，都需要依賴 IP 位址與 MAC 位址的協同運作。而 ARP (Address Resolution Protocol，地址解析協定) 正是負責將網路層的 IP 位址 轉換為資料鏈路層的 MAC 位址的關鍵協定。以下將從 ARP 的運作原理、類型、流程與安全性風險等面向，完整解析 ARP 的概念與應用。

<br />

## ARP 的基本概念

每個網路裝置都有兩種重要的位址

- IP 位址 (網路層)：用於定位網路中的裝置，類似郵寄地址。

- MAC 位址 (資料鏈路層) ：用於在同一個區域網路 (LAN) 內辨識實體裝置，類似門牌號碼。

當裝置 A 想要將資料傳送給同一區域網路內的裝置 B 時，A 需要知道 B 的 MAC 位址，才能正確將封包送到 B。此時，ARP 就派上用場了。

簡單來說，ARP 將已知的 IP 位址解析為對應的 MAC 位址。

<br />

## ARP 的運作原理

ARP 是一種運作在網路層與資料鏈路層之間的協定。

運作原理可概括為以下幾個步驟

1. ARP 請求 (ARP Request)

    - 當裝置 A 需要查詢某個 IP 位址的 MAC 位址時，裝置 A 會在 LAN 上廣播一個 ARP 請求封包。

    - 封包內容會包含

        - 目標 IP 位址

        - 來源 IP 位址

        - 來源 MAC 位址

    - 廣播的意思是「整個區域網路的所有裝置都能收到這個訊息」。

2. ARP 回應 (ARP Reply)

    - 網路中具有目標 IP 的裝置會回應 ARP 回應封包，告訴請求者自己的 MAC 位址。

    - 回應通常是單播，直接發送給請求者。

    - 接收到回應後，裝置 A 就能在自己的 ARP 快取表 (ARP Cache) 中紀錄這個 IP 與 MAC 的對應關係，方便下次直接使用。

A 發送 ARP Request → LAN 廣播 → B 收到並回應 ARP Reply → A 更新快取表。

## ARP 的類型

ARP 主要有以下幾種常見形式

1. 標準 ARP (Standard ARP)

    - 最基本的 IP 到 MAC 對應查詢方式，適用於 IPv4 網路。

2. Proxy ARP (代理 ARP)

    - 當裝置位於不同子網路時，路由器可以代替目標裝置回應 ARP 請求，讓網路裝置以為在同一個區域網路中。

3. Gratuitous ARP (自動通告 ARP)

    - 裝置主動廣播自己的 IP 與 MAC 對應資訊，常用於：

        - IP 衝突檢測

        - 快取更新

- Proxy ARP：在某些子網路隔離環境中，使舊設備仍能互通。  

- Gratuitous ARP：常用於 DHCP 或高可用系統檢測 IP 衝突。

<br />

## ARP 快取表 (ARP Cache)

為了減少每次傳送封包都進行廣播查詢，大多數作業系統會將 IP-MAC 對應關係暫存於 ARP 快取表中。

特性包括

- 動態更新：自動紀錄 ARP 請求與回應。

- 固定表條目：網路管理員可手動設定靜態對應，防止快取被覆蓋。

- 有效時間：動態條目會在一段時間後自動失效，避免資料過時。

<br />

## ARP 的安全性風險

ARP 基於 LAN 廣播信任機制，因此存在安全漏洞，常見的攻擊手法包括

- ARP 欺騙 (ARP Spoofing)

    - 攻擊者偽造 ARP 回應，使裝置 A 誤以為攻擊者是目標裝置。

    - 可能導致中間人攻擊 (MITM) 或封包竊聽 (可以搭配 VLAN 隔離或使用 VPN 加密內部流量來防止 MITM)。

- ARP 洪水 (ARP Flooding)

    - 大量虛假 ARP 封包充斥網路，導致快取混亂或網路阻塞。

防護方式

- 啟用靜態 ARP 表，或使用網路存取控制。

- 使用交換器安全功能 (例如：Dynamic ARP Inspection，簡稱：DAI)。

- 配合加密傳輸協定 (例如：HTTPS) 減少敏感資料暴露風險。

<br />

## ARP 與網路管理

ARP 不僅是網路層與資料鏈路層的橋樑，也常被網管工具用來

- 掃描區域網路設備

- 查詢裝置 MAC 位址

- 進行網路故障排除 (例如：ping 失敗時確認 IP 對應)

在實務操作中，使用 `arp -a` (Windows/Linux/macOS) 即可查看本機 ARP 快取表。

<br />

## 總結

ARP 在網路通訊中扮演著不可或缺的角色

- 負責將 IP 位址轉換為 MAC 位址，確保資料在區域網路內正確傳送。

- 廣播機制雖簡單高效，但也容易受到 ARP 欺騙與快取污染攻擊。

- 透過靜態設定、網路防護功能及加密協定，可以提升 ARP 運作的安全性。

ARP 雖然在背後默默運作，但對網路通訊的順暢與安全具有關鍵影響。
