# Nginx

Nginx (發音為「Engine X」) 是一個開源的高效能網頁伺服器和反向代理伺服器，最初由俄羅斯程式設計師 Igor Sysoev 於 2004 年開發，最初是為了解決 C10k 問題，也就是在同一時間面對高並發情境下的性能瓶頸問題。

<br />

## Nginx 的架構

Nginx 採用了非同步、事件驅動的架構，與傳統的基於進程或執行緒的伺服器架構截然不同。Nginx 使用了一種叫做「反應器」的設計模式，使其能夠同時處理數以萬計的連接，而不需要為每個連接創建新的進程或執行緒。

非同步與事件驅動

- 非同步處理：Nginx 能夠在不阻塞的情況下處理多個連接，這樣的設計可以減少伺服器資源的消耗，提升處理效率。

- 事件驅動：使用事件驅動的機制來管理連接和請求，也就是說，伺服器可以立即響應到達的請求而不需要等待前一個請求完成。

這種架構使得 Nginx 非常適合用於高並發的場景，例如：大型網站、即時通訊應用等。

## Nginx 的核心功能

- 網頁伺服器

    Nginx 作為網頁伺服器，能夠高效服務靜態內容，例如：HTML、CSS 和 JavaScript 等。其優化的靜態內容處理能力使其在提供靜態文件的速度和效率上超過了許多傳統伺服器。

- 反向代理伺服器

    作為反向代理伺服器，Nginx 可以將客戶端的請求轉發到後端的伺服器集群，從而實現負載平衡、流量控制和安全保護。

    - 負載平衡：將流量分配到多個後端伺服器，提高服務的可用性和響應速度。

    - 緩存功能：通過緩存機制加速請求響應，減少後端伺服器的負載。

    - 安全性：隔離客戶端和後端伺服器，隱藏後端伺服器的真實 IP 地址，增強安全性。

- 電子郵件代理

    Nginx 還能用作電子郵件代理伺服器，支持 IMAP、POP3 和 SMTP 協議，提供安全、可靠的電子郵件服務。

- HTTP 加速器

    通過緩存靜態內容和動態內容，Nginx 可以作為 HTTP 加速器來減少服務響應時間。

- SSL/TLS 終止

    Nginx 可以處理 SSL/TLS 連接，將加密的 HTTPS 請求解密並轉發給後端伺服器，減輕後端伺服器的加密負擔。

- 虛擬主機

    Nginx 支持虛擬主機配置，允許在同一個伺服器上處理多個網站，這些網站可以共享相同的 IP 位址或者使用不同的域名。

- 動態模塊

    Nginx 允許通過動態模塊擴展其功能，從而滿足不同場景的需求，例如：gzip 壓縮、Hook、URL 重寫等。

## 常見的 Nginx 應用範圍

- 高流量網站

    Nginx 廣泛應用於高流量網站，例如：WordPress、GitHub、Netflix 等。由於其高效的處理能力和穩定性，許多大規模網站選擇 Nginx 作為前端伺服器來處理大量的並發連接。

- 微服務架構

    在微服務架構中，Nginx 經常用作 API Gateway，負責路由請求、認證和限流等操作，能夠有效管理和分發 API 請求，提高微服務的靈活性和可擴展性。

- 內容傳遞網路 (CDN)

    Nginx 可用於搭建 CDN 網絡，用來高效分發靜態資源 (例如：圖片、影片、文件等)，以減少主伺服器的負載並加速內容交付。

- 反向代理和負載均衡

    Nginx 通常用於反向代理和負載均衡的應用中，可以將流量分配到多個後端伺服器，提供更高的可靠性和性能。

## Nginx vs Apache

Nginx 和 Apache 是兩種最流行的網頁伺服器，各自有著不同的設計理念和應用範圍。

- 架構差異：

    - Nginx：非同步事件驅動，適合高並發情境。

    - Apache：基於進程/執行緒，靈活性高，但在高並發下性能不及 Nginx。

- 資源使用：

    - Nginx：低資源消耗，記憶體和 CPU  使用效率高。

    - Apache：資源消耗與並發數量成正比。

- 擴展性：

    - Nginx：支援動態模組，載入和移除方便。

    - Apache：擁有豐富的模組庫，擴展功能強大。

## Nginx 的優缺點

### 優點

- 高效能與高可擴展性

    - 非同步架構：Nginx 的非同步事件驅動架構使其能夠高效處理大量的並發連接，不會受到傳統同步架構中進程或執行緒切換的影響。

    - 低資源消耗：相比於傳統伺服器，Nginx 在處理大量請求時資源佔用較低，也就是說，能在有限的硬體資源下運行。

- 穩定性

    - 長時間運行：Nginx 在長時間運行下表現出色，具有很高的穩定性和可靠性，因此被許多大型網站和應用程式所使用。

- 負載平衡

    - 高效負載均衡：Nginx 提供了強大的負載均衡功能，可以將流量分配到多個後端伺服器上，提高系統的可用性和擴展性。

- 靈活性

    - 配置簡單：Nginx 的配置文件語法簡單明瞭，易於理解和管理，使伺服器的配置和調整更加便捷。

    - 支持多種協議：除了 HTTP，Nginx 還支持 HTTPS、FTP、SMTP、POP3 和 IMAP 等多種協議，提供多樣化的功能。

- 優化的靜態內容處理

    - 快速服務靜態內容：Nginx 能夠高效提供靜態文件 (例如：圖片、CSS、JavaScript 等)，對於需要快速響應的靜態資源來說非常重要。

- 安全性

    強大的安全性：Nginx 支援 SSL/TLS 加密，能夠有效保護數據傳輸的安全性。此外也可以作為反向代理伺服器，幫助保護後端伺服器直接暴露在網路上的風險。

### 缺點

- 動態內容處理

    動態處理能力有限：雖然 Nginx 在靜態內容和反向代理方面表現出色，但在處理動態內容 (例如：PHP) 方面可能不如 Apache。通常需要將動態處理委託給後端的應用伺服器 (例如：PHP-FPM)。

- 學習曲線

    配置複雜性：雖然 Nginx 的基本配置比較簡單，但對於複雜的需求 (例如：高度自定義的路由規則、反向代理配置等)，配置文件的複雜性可能會增加，對新手來說可能需要一定的學習曲線。

- 模組支援

    動態模組：Nginx 支援動態模組的能力比 Apache 有限，也就是說一些需要特殊功能的模組可能需要重新編譯 Nginx 來支持這些模組。雖然 Nginx 支援載入和移除模組，但某些高級功能可能需要額外的模組支持。

- 社群支持

    社群活躍度：儘管 Nginx 的社群活躍度很高，但與 Apache 相比，擁有的模組和擴展選擇可能稍顯不足。Apache 的長期使用和廣泛應用也意味著有更多的資源和社群支持。

- 診斷與除錯

    日誌和除錯：Nginx 的日誌系統雖然功能強大，但在某些複雜的除錯情況下，可能需要額外的配置或工具來獲取所需的詳細資訊。
